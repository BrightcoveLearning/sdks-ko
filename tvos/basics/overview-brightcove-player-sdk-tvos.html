페이지 내용

    개요: tvOS용 브라이트코브 플레이어 SDK

    tvOS 개발자 설명서용 브라이트코브의 플레이어 SDK에 오신 것을 환영합니다. 여기에서 Apple TV에서 비디오 콘텐츠를 재생하는 방법에 대한 정보를 찾을 수 있습니다.

    개요

    tvOS용 네이티브 SDK를 사용하면 Apple TV용 비디오 중심 앱을 만들 수 있습니다.

    • 비디오 클라우드 고객은 비디오 클라우드 라이브러리 콘텐츠를 재생할 수 있습니다.
    • 브라이트코브 플레이어 고객은 인터넷에 액세스 가능한 URL로 비디오를 재생할 수 있습니다.

    콘텐츠를 인제스트할 때는 Apple TV에 가장 적합한 변환을 위해 고해상도 프로파일을 사용하는 것이 좋습니다. 자세한 내용은 Apple TV용 Apple 비디오 형식설명서를 참조하십시오.

    지원되는 버전

    tvOS용 네이티브 플레이어 SDK는 다음 버전의 tvOS를 지원합니다.

    능동적 지원

    브라이트코브는 다음 tvOS 버전에서 최신 iOS SDK에 대한 적극적인 지원을 제공합니다.

    • 틀: tvOS
    • 틀: tvOS
    • 틀: tvOS

    수동 지원

    브라이트코브는 다음 tvOS 버전에 대한 수동 지원을 제공합니다.

    • 틀: tvOS

    지원 수준 이해

    지원은[주요 버전]의 관점에서 정의됩니다. [ 부 버전] .OS의[패치 버전] (예: tvOS 11.4.1). Apple의 최신 기술 개선 사항을 활용하기 위해 다음과 같은 두 가지 수준의 지원을 제공합니다.

    • 활성 상태 : 브라이트코브는 이러한 OS 버전에서 버그를 테스트하고 수정합니다.
    • 패시브 : 브라이트코브는 이러한 OS 버전에서 테스트하지만 버그 수정은 사례별로 처리됩니다.

    일반적으로 브라이트코브는 사용자 기반의 80% 가 사용하는 OS 버전에 대한 적극적인 지원과 사용자 기반의 5% 이상이 사용하는 다른 버전에 대한 수동 지원을 제공합니다.

    시작하기

    tvOS용 플레이어 SDK로 개발을 시작하기 전에 다음과 같은 몇 가지 개념을 배워야 합니다.

    1. 미디어

      현재 버전의 네이티브 플레이어 SDK에서는 재생 API를 통해 Video Cloud 계정의 콘텐츠를 재생하거나 인터넷에 액세스할 수 있는 URL이 있는 원격 에셋으로 재생할 수 있습니다.

    2. 분석

      비디오 클라우드 고객은 네이티브 플레이어 SDK를 통해 Video Cloud에서 강력한 분석 데이터를 자동으로 얻을 수 있습니다.

    3. 핵심 SDK 및 종속성

      iOS용 브라이트코브 플레이어 SDK 및 해당 플러그인을CocoaPods종속성 관리자를 사용하여 프로젝트에 추가할 수 있습니다. 모든 브라이트코브 플레이어 SDK 구성 요소에 대한 포드스펙은BrightCoveSpecs 리포지토리에서호스팅됩니다.

      CocoaPods 버전 1.0 이상이 필요합니다.
    4. 네이티브 SDK 참조

      SDK에 대한 이해를 돕기 위해 Player SDK에서 아키텍처에 대한 자세한 분석을 찾을 수 있습니다. 주요 구성 요소가 어떻게 상호 작용하여 네이티브 비디오 플레이어 제어의 복잡성을 처리하는지 알아보십시오. 플레이어 SDK 설치, 빠른 시작 및 아키텍처

      SDK의 최신 베타 버전 (있는 경우) 도 여기에서 확인할 수 있습니다.

    5. 질문 및 답변

      SDK 개발자 포럼에참여하여 브라이트코브 엔지니어와 커뮤니티가 SDK 관련 질문에 답변해 드립니다. 이전에 답변한 질문을 검색하거나 새 질문을 게시합니다. 공용 그룹으로서 이메일 주소를 제공하여 액세스 권한을 얻을 수 있습니다.

    SDK 다운로드

    tvOS용 브라이트코브 플레이어 SDK는 iOS용 플레이어 SDK에 포함되어 있습니다. 여기서 tvOS를 지원하는 동적 라이브러리 프레임 워크를 찾을 수 있습니다.

    iOS용 브라이트코브 네이티브 SDK의 최신 버전을 다운로드합니다.

    모든 기본 SDK 및 관련 플러그인에 대한 제품 업데이트를 보려면브라이트코브 네이티브 SDK 문서의 릴리스 노트를참조하십시오.

    사이드카 자막 구성 요소는 캡션을 사용하여 AirPlay를 통한 재생을 지원합니다. 자막을 동기화 상태로 유지하는 방법에 대한 자세한 내용은WebVTT 캡션 동기화문서를 참조하십시오.

    참고:

    • CocoaPods 컴플라이언스를 개선하기 위해 iOS 6.7.7용 네이티브 SDK가 일부 Podspec 파일의 이름을 변경했습니다. 자세한 내용은기본 SDK 참조를참조하십시오.
    • iOS 프레임워크용 모든 브라이트코브 네이티브 플레이어의 최소 배포 대상은 iOS 11.0 및 tvOS 11.0입니다.
    • iOS 6.5.0 이상용 브라이트코브 네이티브 SDK에는 Xcode 11.0 이상이 필요합니다.
    • 프로젝트에서 자동 참조 계수 (ARC) 가 필요합니다. 이것은 Xcode 프로젝트의 현재 기본값입니다.
    • tvOS용 브라이트코브 네이티브 SDK는 Obj-C 및 Swift를 사용하여 맞춤 앱을 만들 수 있도록 작성되었지만 Apple의 TVML (텔레비전 마크업 언어) 은 지원하지 않습니다.
    • FairPlay 스트리밍 콘텐츠의 알려진 Apple 문제는 AirPlay 미러링 모드에서 렌더링되지 않는다는 것입니다. 자세한 내용은WWDC 프레젠테이션을 참조하십시오. HLS에 대한 콘텐츠 보호 (성적 기록의 중간 정도 스크롤)

    통합

    다음 표에는 SDK 구현과 통합된 플러그인을 얻을 수 있는 링크가 포함되어 있습니다. 플러그인은 각각의 Github Repos에서 다운로드 할 수 있으며 각 repo에는 플러그인 설정 방법에 대한 지침과 함께 자세한 README가 포함되어 있습니다.

    샘플 응용 프로그램은 SDK와 통합된 각 플러그인의 기본 기능을 보여줍니다. 이러한 샘플 앱은 각 플러그인에 대해 알아볼 수 있는 좋은 방법이며 디버깅 시 참조점 역할을 합니다.

    iOS용 네이티브 SDK의 현재 릴리스에는 동일한 버전 번호를 가진 모든 구성 요소가 포함되어 있습니다. 구성 요소를 통합할 때 SDK와 구성 요소 버전 번호가 일치해야 합니다. 예를 들어 IMA 플러그인 구성 요소 6.7.0을 사용하는 경우 네이티브 SDK 6.7.0을 사용해야 합니다.

    샘플

    다음 표에는 이전 섹션의 통합과 관련된 샘플 외에 코드 샘플에 대한 링크가 포함되어 있습니다.

    샘플 애플리케이션 설명
    애플 TV Video Cloud 라이브러리에서 비디오를 재생하는 기본 앱을 만듭니다.
    페어플레이로 재생 FPS (페어플레이 스트리밍) 로 콘텐츠를 보호하세요. 전체 샘플은기본 FairPlay iOS 샘플을 참조하십시오.
    사이드카 자막 별도의 파일에 포함된 비디오의 캡션을 표시합니다. 이것은 iOS 샘플이지만 tvOS와 관련이 있어야합니다.

    솔루션 가이드

    다음 표에는 브라이트코브 Player SDK를 사용하여 자신의 앱을 시작하는 데 도움이 되는 개발자 문서에 대한 링크가 포함되어 있으며 사용 가능한 일부 기능과 기능을 안내합니다.

    가이드 설명
    애플 TV UI 컨트롤 작업하기 TVOS용 브라이트코브 네이티브 SDK를 활용하여 TV 플레이어 컨트롤을 사용하는 방법을 알아보십시오.
    간단한 비디오 재생 tvOS용 브라이트코브 네이티브 SDK를 사용하여 간단한 비디오 앱을 만드는 방법을 알아봅니다.
    소스 선택 비디오를 재생할 때 기본 소스 선택 프로세스에 대해 알아봅니다.
    웹VTT 캡션 사용 tvOS용 네이티브 SDK를 사용할 때 HLS 비디오용 WebVTT 캡션을 구성하는 방법에 대해 알아봅니다.
