개요

tvOS 용 네이티브 SDK를 사용하면 Apple TV 용 비디오 중심 앱을 만들 수 있습니다.

  • Video Cloud 고객은 Video Cloud 라이브러리 콘텐츠를 재생할 수 있습니다.
  • Brightcove Player 고객은 인터넷 액세스 URL로 비디오를 재생할 수 있습니다.

콘텐츠를 수집 할 때 Apple TV를위한 최상의 변환을 위해 고해상도 프로필을 사용하는 것이 좋습니다. 자세한 내용은 Apple의 Apple TV 용 비디오 형식선적 서류 비치.

지원되는 버전

적극적인 지원

Brightcove는 다음 tvOS 버전의 최신 공개 릴리스에서 최신 tvOS SDK에 대한 적극적인 지원을 제공합니다.

  • tvOS 14
  • tvOS 13.4.8
  • tvOS 12.4.1

수동적 지원

Brightcove는 다음 tvOS 버전에 대한 수동 지원을 제공합니다.

  • tvOS 11.4.1

지원 수준 이해

지원은 [주 버전] . [부 버전] . [패치 버전] OS (예 : tvOS 11.4.1). Apple의 최신 기술 개선 사항을 활용하기 위해 두 가지 수준의 지원을 제공합니다.

  • 활동적인 : Brightcove는 이러한 OS 버전에서 버그를 테스트하고 수정합니다.
  • 수동적 인 : Brightcove는 이러한 OS 버전에서 테스트하지만 버그 수정은 사례별로 처리됩니다.

일반적으로 Brightcove는 사용자 기반의 80%가 사용하는 OS 버전에 대해 능동 지원을 제공하고 사용자 기반의 최소 5%가 사용하는 다른 버전에 대해서는 수동 지원을 제공합니다.

시작하기

tvOS 용 플레이어 SDK로 개발을 시작하기 전에 알아야 할 몇 가지 개념이 있습니다.

  1. 미디어

    현재 버전의 Native Player SDK는 Playback API를 통해 Video Cloud 계정의 콘텐츠를 재생하거나 인터넷에 액세스할 수 있는 URL이 있는 원격 자산으로 재생하는 것을 지원합니다.

  2. 해석학

    Native Player SDK를 사용하면 Video Cloud 고객은 Video Cloud에서 보고된 강력한 분석 데이터를 자동으로 얻습니다.

  3. 핵심 SDK 및 종속성

    iOS용 Brightcove Player SDK 및 해당 플러그인은 다음을 사용하여 프로젝트에 추가할 수 있습니다. 코코아팟의존성 관리자. 모든 Brightcove Player SDK 구성 요소에 대한 포드 사양은 BrightcoveSpecs 저장소 .

    CocoaPods 버전 1.0 이상이 필요합니다.
  4. 네이티브 SDK 참조

    SDK에 대한 이해를 돕기 위해 플레이어 SDK에서 아키텍처에 대한 자세한 분석을 찾을 수 있습니다. 기본 비디오 플레이어 제어의 복잡성을 처리하기 위해 중요한 구성 요소가 어떻게 상호 작용하는지 알아보십시오. 플레이어 SDK 설치, 빠른 시작 및 아키텍처

    여기에서 SDK의 최신 베타 버전(있는 경우)도 찾을 수 있습니다.

  5. 질문과 답변

    우리와 함께 SDK 개발자 포럼여기에서 SDK 관련 질문은 Brightcove 엔지니어와 커뮤니티에서 답변합니다. 이전에 답변한 질문을 검색하거나 새 질문을 게시하세요. 공개 그룹은 이메일 주소를 제공하여 액세스 권한을 얻을 수 있습니다.

노트:

  • CocoaPods 규정 준수를 개선하기 위해 iOS 6.7.7용 기본 SDK에서 일부 Podspec 파일의 이름이 변경되었습니다. 자세한 내용은네이티브 SDK 참조 .
  • iOS 프레임 워크 용 모든 Brightcove 네이티브 플레이어의 최소 배포 대상은 이제 iOS 11.0 및 tvOS 11.0입니다.
  • iOS 6.5.0+용 Brightcove 네이티브 SDK에는 Xcode 11.0 이상이 필요합니다.
  • 프로젝트에 자동 참조 계산(ARC)이 필요합니다. 이것은 Xcode 프로젝트의 현재 기본값입니다.
  • tvOS 용 Brightcove 네이티브 SDK는 Obj-C 및 Swift를 사용하여 사용자 지정 앱을 만들 수 있도록 작성되었지만 Apple의 TVML (TV 마크 업 언어)을 지원하지 않습니다.
  • FairPlay 스트리밍 콘텐츠의 알려진 Apple 문제는 AirPlay 미러링 모드에서 렌더링되지 않는다는 것입니다. 자세한 내용은 WWDC 프레젠테이션: HLS를 위한 콘텐츠 보호 (스크립트 중간쯤 스크롤).

통합

다음 표에는 SDK 구현과 통합된 플러그인을 가져오는 링크가 포함되어 있습니다. 플러그인은 해당 Github 리포지토리에서 다운로드할 수 있으며 각 리포지토리에는 플러그인 설정 방법에 대한 지침이 포함된 자세한 README가 포함되어 있습니다.

샘플 애플리케이션은 SDK와 통합된 각 플러그인의 기본 기능을 보여줍니다. 이러한 샘플 앱은 각 플러그인에 대해 배울 수 있는 좋은 방법이며 디버깅할 때 참조 지점 역할을 합니다.

iOS용 네이티브 SDK의 현재 릴리스에는 버전 번호가 동일한 모든 구성 요소가 포함되어 있습니다. 구성 요소를 통합할 때 SDK와 구성 요소 버전 번호가 일치해야 합니다. 예를 들어 IMA 플러그인 구성요소 6.7.0을 사용하는 경우 기본 SDK 6.7.0을 사용해야 합니다.
