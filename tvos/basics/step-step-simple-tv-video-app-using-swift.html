Xcode 프로젝트 설정

앱에 대한 새 Xcode 프로젝트를 만듭니다.

  1. Xcode에서 새 tvOS 프로젝트 생성을 시작합니다. 선택하다새 Xcode 프로젝트 생성 .

  2. tvOS 선택단일보기 애플리케이션템플릿에 대해다음 .

  3. 프로젝트 정보를 다음과 같이 설정합니다.

    • 상품명: 단순 비디오 재생
    • 팀:없음

      팀 필드는 선택 사항입니다. 앱 스토어에서 앱을 배포하려면 Apple 개발자 프로그램을 통해 소속된 팀을 선택해야 합니다. 코드 서명에는 팀 이름이 필요합니다. 이 예에서는 없음을 선택합니다.

    • 조직 이름:당신의 회사 이름
    • 조직 식별자: com.example-company
      이렇게 하면 앱 스토어에서 제품 이름이 고유해집니다.
    • 언어: 빠른

  4. 프로젝트를 저장하고 생성할 위치를 선택합니다.

  5. 이제 프로젝트를 닫습니다( 예, 닫습니다. 이것은 중요합니다! )

프로젝트에 SDK 및 해당 종속성 추가

SDK 및 해당 종속성을 프로젝트에 추가하는 가장 쉬운 방법은 CocoaPods를 사용하는 것입니다.

CocoaPods는 프로젝트에 라이브러리를 추가하는 종속성 관리자입니다. 필수는 아니지만 설치가 더 쉽습니다. CocoaPods를 설치하려면 설명서의 지침을 참조하십시오. 코코아팟대지.

  • CocoaPods 버전 1.0 이상이 필요합니다.
  • CocoaPods( pod도구) 기존 Xcode 프로젝트에서 프로젝트 파일을 수정합니다. 이 작업을 수행하기 전에 백업을 만들 수 있습니다.
To easily create and update the Podfile, you can do the following steps inside of the CocoaPods App.

  1. 프로젝트 폴더에서 이라는 일반 텍스트 파일을 만듭니다. 팟파일 (파일 확장자 없음).

  2. 텍스트 편집기를 사용하여 다음 코드 줄을 Podfile에 추가하고 저장합니다. 이 코드는 다음을 수행합니다.

    • 1행: GitHub 소스를 정의합니다.
    • 3행: 정적 라이브러리 대신 프레임워크를 사용하도록 Pod를 설정합니다.
    • 5행: tvOS 플랫폼 버전을 정의합니다.

    • 7-9행: Brightcove 네이티브 플레이어 SDK를 설치합니다.

      1. 에서 현재 네이티브 SDK 버전을 가져옵니다. iOS용 Brightcove 기본 SDK 개요문서.
      2. 값 바꾸기 '6.3.11'현재 SDK 버전 번호로.

      3. 종속성 관리에 대한 자세한 내용은 다음을 참조하십시오. CocoaPods로 종속성을 관리하는 방법기사.

    source 'https://github.com/brightcove/BrightcoveSpecs.git'

use_frameworks!

    • NS target속성은 CocoaPods 버전 1.0 이상에 사용됩니다.
    • UI 컨트롤을 지원하려면 Brightcove 네이티브 SDK 코어 v6.3 +를 대상으로해야합니다.

  3. 터미널 세션을 열고단순 비디오 재생 Xcode 프로젝트 폴더.

  4. 터미널 세션에서 다음 명령을 입력하십시오.

    pod install

    그리고 누르다반품실행합니다. Brightcove Player SDK가 프로젝트에 추가되었음을 나타내는 일련의 메시지가 터미널에 표시되어야 합니다.

  5. 마지막 줄을 주목하라 , 중요합니다. 이 시점부터 Simple-Video-Playback.xcworkspace Xcode의 파일, ~ 아니다 NS Simple-Video-Playback.xcodeproj 파일. NS pod install명령이 이것을 생성합니다.xcworkspace프로젝트에 대한 파일입니다.

오디오 세션은 앱 수준에서 오디오 동작을 처리합니다. 에 대해 자세히 알아보기 AV오디오세션수업.

미디어 보안 관리

ATS(앱 전송 보안)는 앱과 웹 서비스 간에 보안 연결을 적용합니다. Apple의 iOS 9 SDK 릴리스와 함께앱 전송 보안(ATS)추가되었습니다.

  1. 다음 상황 중 하나가 앱에 적용됩니다.

    • 기본적으로 iOS용 Brightcove 기본 SDK는 소스 선택 정책을 사용하여 HTTP 소스보다 HTTPS를 선택하므로 ATS가 활성화된 앱을 빌드할 수 있습니다.

      이제 앱을 실행할 준비가 되었습니다.


    • HTTP 소스를 사용 중이거나 앱에 다른 HTTP 호출이 있는 경우 다음 오류 메시지가 표시될 수 있습니다.

      App Transport Security has blocked a cleartext HTTP (http://) resource load since it is insecure.
Temporary exceptions can be configured via your app's Info.plist file.

      이는 ATS가 활성화되었지만 시스템이 ATS 요구 사항을 충족하도록 구성되지 않았음을 의미합니다. 이 상황을 해결하려면 다음을 참조하십시오. 앱 전송 보안(ATS) 작업문서.
