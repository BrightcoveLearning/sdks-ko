지원 수준 이해

  • 활동적인 : Brightcove는 이러한 OS 버전에서 버그를 테스트하고 수정합니다.

일반적으로 Brightcove는 사용자 기반의 80%가 사용하는 OS 버전에 대해 능동 지원을 제공하고 사용자 기반의 최소 5%가 사용하는 다른 버전에 대해서는 수동 지원을 제공합니다.

시작하기

Android SDK로 개발을 시작하기 전에 알아야 할 몇 가지 개념이 있습니다.

  1. 미디어

    현재 버전의 Native Player SDK는 Playback API를 통해 Video Cloud 계정의 콘텐츠를 재생하거나 인터넷에 액세스할 수 있는 URL이 있는 원격 자산으로 재생하는 것을 지원합니다.

  2. 해석학

    Native Player SDK를 사용하면 Video Cloud 고객은 Video Cloud에서 보고된 강력한 분석 데이터를 자동으로 얻습니다.

  3. 그라들

    Gradle은 Android SDK 빌드 시스템의 기반이며 Android Studio와 통합됩니다.

  4. 메이븐 레포

    이제 Brightcove Maven Repo에서 Android Maven 아티팩트(jar 파일)용 Brightcove 기본 플레이어를 사용할 수 있습니다. https://repo.brightcove.com/releases

    Android 프로젝트에 Maven 저장소를 추가하는 방법에 대한 자세한 내용은 Android용 네이티브 SDK를 사용하여 앱 빌드문서.
  5. Android SDK 개요

    SDK에 대한 이해를 돕기 위해 기본 비디오 플레이어 제어의 복잡성을 처리하기 위해 중요한 구성 요소가 상호 작용하는 방법을 알아보세요.

  6. Android용 네이티브 SDK를 사용하여 앱 빌드

    자체 백엔드 시스템에서 비디오를 재생하는 Android용 Brightcove Player SDK로 첫 번째 앱을 만드십시오. Video Cloud 고객은 추가로 Brightcove 카탈로그 서비스에서 비디오를 검색하고 재생할 수 있습니다.

  7. SDK 개발자 포럼

    Brightcove 엔지니어 및 커뮤니티에서 SDK 관련 질문에 대한 답변을 제공하는 SDK 개발자 포럼에 참여하십시오. 이전에 답변한 질문을 검색하거나 새 질문을 게시하세요. 공개 그룹은 이메일 주소를 제공하여 액세스 권한을 얻을 수 있습니다.

  8. 주문형 비디오 교육

    Video Cloud 라이브러리에서 비디오와 재생 목록을 재생하는 Android Studio에서 앱을 빌드하는 단계를 안내하는 일련의 교육 비디오를 보십시오.

  • Android용 Brightcove 네이티브 SDK에서 지원하는 Android 최소 SDK 버전은 이제 19입니다.
  • ExoPlayer v2.9.0+의 경우 Java 8 지원을 활성화해야 합니다. 이렇게하려면 다음을 추가하십시오. android앱의 섹션 build.gradle파일. 자세한 내용은안드로이드의 엑소플레이어문서.

핵심 SDK 설치

Brightcove 핵심 SDK 설치는 매우 간단합니다. 두 가지 방법이 있습니다.

  • Gradle 빌드 시스템 사용
  • SDK 수동 다운로드

Gradle 빌드 시스템 사용

Brightcove SDK 및 해당 종속성을 추가하는 가장 쉬운 방법은 Android Studio에서 gradle 빌드 시스템을 사용하는 것입니다.

'+와 함께 동적 종속성 버전 사용' 캐릭터는 Gradle 커뮤니티에서 권장되지 않습니다. 동적 버전 관리를 사용하면 최신 API가 앱의 소스 코드와 예기치 않게 호환되지 않을 수 있으므로 빌드 프로세스에 상당한 위험이 따릅니다.

Brightcove는 항상 특정 버전 문자열과 함께 최신 버전을 사용할 것을 권장합니다. 예를 들어, build.gradle그리고 gradle.properties파일안드로이드 플레이어 샘플프로젝트.

Gradle 사용에 대한 자세한 내용은 Android의빌드 종속성 추가선적 서류 비치.

SDK 수동 다운로드

다음과 같이 Brightcove SDK를 수동으로 다운로드할 수도 있습니다.

  1. SDK를 다운로드하고 압축을 풉니다. 여러 폴더를 찾을 수 있습니다.
  2. 내부 SDK폴더에서 이름을 가진 핵심 SDK를 찾을 수 있습니다. android-sdk-xxxaar , 어디트리플 엑스최신 SDK 버전 번호입니다.
  3. 추가 android-sdk-xxxaar필요한 플러그인 파일과 함께 라이브러리에 파일을 추가합니다.

모범 사례

다음은 Brightcove Native SDK로 작업할 때 도움이 될 수 있는 추가 정보입니다.

IMA로 광고하기

VAST 광고의 배치는 일반적으로 VMAP 문서에 의해 지정됩니다. VAST는 광고를 설명하고 VMAP은 동영상의 광고 게재위치를 설명합니다. 코드 세부 정보는 다음을 참조하세요.

  • AdRulesIMASampleApp
  • AdRulesIMAWidevineModularSampleApp

  • 기본IMA샘플앱

    기본 IMA 샘플의 경우 다음을 변경하는 것이 좋습니다. BrightcoveVideoView 에게BrightcoveExoPlayerVideoView , ExoPlayer 프레임워크를 사용합니다.

 

IMA 광고로 작업할 때 광고를 구현하기 전에 광고가 SDK 외부에서 작동하는지 확인하는 것이 가장 좋습니다. 다음 링크가 도움이 될 것입니다.

  • Google VAST 광고 응답 검사기
  • 디지털 비디오 다중 광고 재생 목록(VMAP) 가이드
  • Google IMA 기술 포럼

컴패니언 광고

컴패니언 광고 및 클릭연결 광고는 Android TV에서 지원되지 않습니다. Android TV에는 웹 브라우저가 지원되지 않으므로 클릭연결 URL의 대상이 없습니다.

ProGuard 작업

사용프로가드더 안전한 앱을 만들 수 있으므로 리버스 엔지니어링이 더 어렵습니다. ProGuard는 Android 빌드 시스템에 통합되어 있으므로 활성화하거나 없이 빌드할 수 있습니다.

Gradle에서 ProGuard가 활성화되었을 때 오류가 발생하면 다음 제외 규칙을 추가해 보세요.

  -dontwarn com.google.**
  -dontwarn android.media.**
  -keep class android.media.** { *; }
  -keep class com.google.** { *; }
  -keep interface com.google.** { *; }
  -keep class com.google.ads.interactivemedia.** { *; }
  -keep interface com.google.ads.interactivemedia.** { *; }

Brightcove Native SDK에서 사용하는 ProGuard 규칙의 전체 목록을 보려면 다운로드하여 압축을 풉니다. Android용 현재 네이티브 SDK . 열기 SDK폴더 및 보기proguard.txt 파일.
