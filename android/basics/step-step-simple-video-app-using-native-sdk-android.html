Gradle을 활용하여 Android용 Brightcove Player SDK에 연결

Gradle은 Android Studio와 통합되어 있으므로 이를 사용하여 Android용 네이티브 SDK를 프로젝트에 추가합니다.

  1. 확장 Gradle 스크립트섹션을 열고 두 번째build.gradle PlayVideos 프로젝트의 현재 모듈과 연결된 파일입니다.

  2. 에서build.gradle파일에서 다음을 수행합니다.

    1. 에서android섹션에 다음을 추가하십시오. compileOptions값:
      ExoPlayer v2.9.0+의 경우 Java 8 지원을 활성화해야 합니다. 이렇게하려면 다음을 추가하십시오. android앱의 섹션 build.gradle파일. 자세한 내용은안드로이드의 엑소플레이어문서.

        minSdkVersion 21
        targetSdkVersion 28
        versionCode 1
        versionName "1.0"
        testInstrumentationRunner "androidx.test.runner.AndroidJUnitRunner"
    }
    buildTypes {
        release {
            minifyEnabled false
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }

        targetCompatibility JavaVersion.VERSION_1_8

    노트:
    • Brightcove는 항상 특정 버전 문자열과 함께 최신 버전을 사용할 것을 권장합니다. 예를 들어, build.gradle그리고 gradle.properties파일안드로이드 플레이어 샘플프로젝트.
    • ExoPlayer 종속성은 Android용 Native SDK를 포함하므로 SDK를 별도로 포함할 필요가 없습니다.
    • 구글 엑소플레이어 Android용 오픈 소스 애플리케이션 수준 미디어 플레이어입니다. ExoPlayer 2 프레임워크를 사용하고 있습니다. 자세한 내용은 ExoPlayer 2 프레임워크로 마이그레이션문서.
  3. 프로젝트를 동기화하여 Gradle 파일 변경 사항을 선택합니다.

  @Override

      brightcoveVideoView.add(video);
      brightcoveVideoView.start();
    }

  6. 에서onError()메서드에서 오류 문자열을 throw합니다.

    @Override

  7. 귀하의 완전한 코드MainActivity클래스는 다음과 유사해야 합니다.

    public class MainActivity extends BrightcovePlayer {

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);

    EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();
    Catalog catalog = new Catalog(eventEmitter, getString(R.string.account), getString(R.string.policy));
    catalog.findVideoByID(getString(R.string.videoId), new VideoListener() {

      // Add the video found to the queue with add().
      // Start playback of the video with start().
      @Override
      public void onVideo(Video video) {
        brightcoveVideoView.add(video);
        brightcoveVideoView.start();
      }

      @Override
      public void onError(String s) {
        throw new RuntimeException(s);
      }
    });
  }
}

NS com.brightcove.player.edge.Catalog클래스는 비디오 및 재생 목록을 검색하기 위한 비동기 메서드를 제공합니다. 브라이트코브 재생 API . 이것은 Video Cloud 라이브러리에서 콘텐츠를 검색하기 위한 최신 권장 API입니다.

불필요한 코드 제거

  1. 이전 앱의 일부 코드는 더 이상 필요하지 않습니다. 카탈로그에 대한 호출을 제거합니다. findVideoByID()방법 및 관련VideoListener익명 콜백 함수.

    catalog.findVideoByID(getString(R.string.videoId), new VideoListener() {

    // Add the video found to the queue with add().
    // Start playback of the video with start().
    @Override
      public void onVideo(Video video) {
        brightcoveVideoView.add(video);
        brightcoveVideoView.start();
      }

    @Override
      public void onError(String s) {
        throw new RuntimeException(s);
      }
  });

카탈로그에서 재생 목록 검색

  1. 기존 카탈로그 인스턴스는 재생 목록을 검색하는 데 작동하므로변경이 필요하지 않습니다다음 코드 줄에:

    EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();
Catalog catalog = new Catalog(eventEmitter, getString(R.string.account), getString(R.string.policy));
  2. 비디오 클라우드 스튜디오에서미디어모듈에서 재생 목록을 선택하고재생 목록 ID .

  3. 카탈로그 사용findPlaylistByID()재생 목록 ID와PlaylistListener콜백을 위해.

    catalog.findPlaylistByID("2764931905001", new PlaylistListener() {
  @Override
  public void onPlaylist(Playlist playlist) {
  }

  @Override
  public void onError(String s) {
  }
});
    NS PlaylistListener비동기 작업에서 재생 목록 개체를 수신할 것으로 예상되는 개체를 식별합니다. 성공적인 수술에는onPlaylist()메소드가 호출됩니다. 작업에 오류가 있는 경우, onError()메소드가 호출됩니다. Android Studio는 필요한 두 가지 메서드의 서명을 삽입했습니다.

  4. 에서onPlaylist()방법, 재생 목록에서 비디오 검색, 모든 비디오 추가brightcoveVideoView , 첫 번째 비디오를 시작합니다.

    @Override
    public void onPlaylist(Playlist playlist) {
      brightcoveVideoView.addAll(playlist.getVideos());
      brightcoveVideoView.start();
    }

  5. 에서onError()메서드에서 오류 문자열을 throw합니다.

     @Override
    public void onError(String s) {
      throw new RuntimeException(s);
    }

  6. 귀하의 완전한 코드MainActivity클래스는 다음과 유사해야 합니다.

    public class MainActivity extends BrightcovePlayer {

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);

    EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();
    Catalog catalog = new Catalog(eventEmitter, getString(R.string.account), getString(R.string.policy));
    catalog.findPlaylistByID("4845949311001", new PlaylistListener() {
      @Override
      public void onPlaylist(Playlist playlist) {
        brightcoveVideoView.addAll(playlist.getVideos());
        brightcoveVideoView.start();
      }

      @Override
      public void onError(String s) {
        throw new RuntimeException(s);
      }
    });
  }
}
