소개

Android용 기본 SDK는 비디오 플레이어로 사용자 작업을 수신하는 이벤트 기반 모델을 사용합니다.

이벤트 기반 모델

기존 프로그래밍 모델에서 흐름은 정적이며 프로그래머가 미리 정의합니다. 이벤트 기반 모델에는 미리 설정된 흐름이 없습니다. 대신 흐름은 사용자 작업과 같은 이벤트에 의해 결정됩니다. 일반적으로 이벤트를 수신하고 콜백 기능을 트리거하는 메인 루프가 있습니다.

이벤트

Android용 Brightcove Player SDK를 사용하는 동안 미리 정의된 이벤트를 활용하여 앱의 기능을 제어할 수 있습니다. 이벤트는 문자열 리터럴일 뿐이므로 고유한 이벤트를 쉽게 만들고 사용할 수 있습니다.

Android용 Brightcove Player SDK의 이벤트는 리스너 체인을 통해 전달될 수 있는 상태 또는 작업을 나타냅니다. 모든 이벤트에는 유형과 연관된 속성 맵이 있습니다. 이벤트가 리스너 체인을 따라 전파되는 것을 중지하거나 비디오 재생 클래스에서 기본 리스너를 방지하도록 지시할 수 있습니다.

이벤트는 발생하는 상태 또는 동작을 나타낼 수 있습니다. 예를 들어 이벤트 유형DID_SEEK_TO 동작의 상태를 나타냅니다SEEK_TO .

사전 정의된 이벤트의 전체 목록을 보려면 다음 이미지를 선택하십시오.

이벤트 이미터

브라이트코브 BaseVideoView Android의 기본비디오 보기 . 이 수업도 시작됩니다이벤트 이미터모든 SDK 구성 요소를 함께 연결합니다. 사용자가 동영상 보기와 상호작용할 때EventEmitter 재생, 일시 중지 등과 같은 이벤트를 발생시킵니다. 각 이벤트에 관심이 있는 구성 요소는 해당 이벤트를 수신하고 조치를 취할 수 있습니다.

SDK가 특정 작업을 수행하기를 원할 때 이벤트를 수신하고 내보내는 플러그인을 추가할 수 있습니다.

네이티브 SDK는 앱을 제어하기 위해 미리 정의된 이벤트를 제공하지만, 자신의 이벤트를 쉽게 만들고 사용할 수 있습니다.

자신의 이벤트 이미터를 만들고 지시하는 것도 가능합니다. BrightcoveExoPlayerVideoView그것을 사용합니다. 이것은 리스너 체인을 제어하려는 경우에 유용합니다.

구성 요소 및 이벤트

Android용 Brightcove SDK의 각 구성 요소에 대해 연결된 모든 이벤트를 찾을 수 있습니다. 이렇게 하려면 다음 단계를 따르세요.

  1. 다음으로 이동합니다. Android용 Brightcove 플레이어 SDK참조.
  2. 왼쪽 메뉴에서 구성 요소를 선택합니다.
  3. 구성 요소 클래스 세부 정보 섹션에서 이 구성 요소에서 내보내고 수신하는 이벤트 목록을 찾을 수 있습니다.

앱에서 Android용 Brightcove Player SDK의 광고 이벤트 세트를 수신하고 이에 따라 조치를 취할 수 있습니다.

IMA 광고 이벤트

이 다이어그램은 Google IMA 광고 플러그인에 대한 광고 이벤트를 보여줍니다. 다음 광고 이벤트는 exoplayer 샘플 내에서 수신됩니다. AdRulesIMASampleApp .

일반적인 광고 이벤트

다음은 수신 대기하고 조치를 취할 수 있는 Android용 Brightcove Player SDK의 일반적인 광고 이벤트 목록입니다.

  • AD_BREAK_STARTED
  • AD_BREAK_COMPLETED
  • AD_STARTED
  • AD_PAUSED
  • AD_RESUMED
  • AD_PROGRESS
  • AD_COMPLETED
  • AD_ERROR

코드 예

광고 시간이 시작되면EventType.AD_BREAK_STARTED이벤트가 발생합니다. 광고 시간이 끝나면EventType.AD_BREAK_COMPLETED방출됩니다.

다음과 같이 이벤트를 수신하고 반응할 수 있습니다.

eventEmitter.on(EventType.AD_BREAK_STARTED, new EventListener() {
	@Override
	public void processEvent(Event event) {
		// React to the event
	}
});

eventEmitter.on(EventType.AD_BREAK_COMPLETED, new EventListener() {
	@Override
	public void processEvent(Event event) {
		// React to the event
	}
});

다음을 수행하여 이벤트를 내보낼 수 있습니다.

eventEmitter.emit(EventType.STOP);
