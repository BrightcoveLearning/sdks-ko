BrightcovePlayer 클래스

NS 브라이트코브플레이어클래스는 Android용 핵심 Brightcove SDK에 있으며 Android의활동수업.

기본으로Activity , NS BrightcovePlayer클래스 랩BrightcoveExoPlayerVideoView그리고 설정EventLogger그리고 기본MediaController . 기본 수명 주기 관리 및 전체 화면 로직을 구현합니다. 하위 분류를 의미합니다.

public class MainActivity extends BrightcovePlayer {
...
}

최소한onCreate()메서드를 연결하려면 논리로 재정의해야 합니다. BrightcoveExoPlayerVideoView변수레이아웃.xml파일. 예를 들면 다음과 같습니다.

brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);

구현에 대한 자세한 내용은 다음을 참조하십시오. 기본샘플앱코드 샘플.

지원 라이브러리 구성 요소

사용하지 않기로 선택한 경우BrightcovePlayer수업, BrightcovePlayerActivity그리고BrightcovePlayerFragment수업이 가능합니다. 이 클래스 중 하나를 사용하려면 다음을 포함해야 합니다. appcompat플러그인 build.gradle파일을 다음과 같이

implementation "com.brightcove.player:android-appcompat-plugin:${anpVersion}"

BrightcovePlayerFragment 클래스

NS BrightcovePlayerFragment클래스 확장 android.support.v4.app.Fragment수업.

Brightcove Player를 레이아웃의 다른 디자인 요소와 결합하려면 다음을 사용할 수 있습니다. BrightcovePlayerFragment수업. 이는 UI 구성 요소를 결합할 공간이 더 많은 태블릿 또는 더 큰 화면용으로 디자인할 때 유용합니다. 예를 들어 태블릿을 사용하면 동일한 활동 내에 있는 여러 조각을 동시에 표시할 수 있습니다.

public class MainFragment extends BrightcovePlayerFragment {
...
}

NS onCreateView()프래그먼트의 레이아웃을baseVideoView호출 전 인스턴스 변수super.onCreateView() .

baseVideoView = (BaseVideoView) result.findViewById(R.id.brightcove_video_view);

구현에 대한 자세한 내용은 다음을 참조하십시오. AppCompatFragmentSampleApp코드 샘플.

수명 주기 관리

다음 중 하나로 시작하는 것이 가장 좋습니다. BrightcovePlayer활동 또는BrightcovePlayerFragment , 이는 다음을 포함하여 기본 수명 주기 관리 구현을 제공하기 때문입니다.

  • 사용자가 앱을 일시 중지하면 클래스는 사용자가 중지한 위치를 추적합니다.
  • 앱이 백그라운드에서 돌아올 때 클래스는 비디오가 재생 중인지 여부를 기억하고 중단된 부분부터 재생을 다시 시작합니다.
  • 메모리 문제로 인해 앱이 종료되면 클래스는 비디오에서 원래 위치로 시간을 찾습니다.
  • 모든 이벤트를 logcat에 기록하는 이벤트 로거가 설정됩니다. 이것은 디버깅에 도움이 될 수 있습니다.
  • 클래스는 플러그인과 통신하기 위해 이벤트를 전달합니다.
 

이제 활동이 있으므로 다음 문서에 설명된 대로 비디오 보기를 선택할 준비가 되었습니다.
