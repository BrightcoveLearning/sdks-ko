모바일 장치에서 비디오 콘텐츠를 다운로드하고 연결 없이 시청하려는 수요가 증가함에 따라 사용자의 시청 행동을 이해하는 능력이 어려울 수 있습니다.

Android 및 iOS용 Brightcove Native Player SDK를 사용하면 오프라인 재생과 관련된 분석을 제공하여 사용자 행동을 추적하고 학습할 수 있습니다.

개요

오프라인 재생과 관련된 분석 데이터에는 비디오 다운로드 및 오프라인 재생 측정항목이 모두 포함됩니다. Native Player SDK는 이러한 측정항목을 다음과 같이 처리합니다.

  • 다운로드 데이터는 비디오 다운로드 중에 Brightcove로 전송됩니다.
  • 오프라인 재생 데이터는 네트워크에 연결될 때까지 장치에 로컬로 저장됩니다.
  • 앱 개발자는 SDK가 모바일 연결을 통해 데이터를 업로드하도록 하거나 업로드를 WIFI로만 제한하도록 선택할 수 있습니다.
  • 저장되는 데이터 양의 기본 한도는 500MB로 설정되지만 앱 개발자는 이 설정을 재정의할 수 있습니다. 이것은 수백 시간의 오프라인 재생 메트릭에 해당합니다.
  • 모든 이벤트에 대해 재시도 로직이 존재하지만, 우리는 비디오 조회를 우선시하고 이를 먼저 전송합니다.

동영상 다운로드 데이터

측정항목에는 다음이 포함됩니다.

  • 변환 비트 전송률 및 변환 크기를 포함한 다운로드 요청
  • 다운로드를 완료하였습니다
  • 다운로드 실패
  • 다운로드 취소

분석 보기

사용애널리틱스 API , 다음 다운로드 측정항목을 볼 수 있습니다.

치수 들
동영상 video_download_request
  video_download_complete
  video_download_cancellation
  video_download_error

Analytics API에서 이러한 다운로드 측정항목을 보는 방법에 대한 자세한 내용은차원: 비디오문서.

샘플 요청

다운로드 관련 필드에 대한 데이터를 반환하려면HTTP GET에 요청애널리틱스 API다음과 유사하게 보일 것입니다:

https://analytics.api.brightcove.com/v1/data?accounts= 귀하의 계정 ID &dimensions=video&fields=video_download_request,video_download_complete,video_download_cancellation,video_download_error

오프라인 재생 데이터

측정항목에는 다음이 포함됩니다.

  • 장치 유형, 제조업체, OS, OS 버전
  • 지리적 위치
  • 도메인
  • 동영상 ID
  • 동영상 조회수(오프라인 동영상 조회수)
  • 동영상 참여(본 동영상의 비율)
