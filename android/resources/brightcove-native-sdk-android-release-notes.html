현재 릴리스

버전 6.16.3

2021년 7월 1일

SDK 변경 사항:

  • 레이블 속성 설정에 대한 지원이 추가되었습니다. 브라이트코브캡션포맷선택한 캡션 및 오디오 트랙 정보를 Cast 수신기로 보내는 것을 지원하는 클래스
  • 업데이트 VideoUtils Video 객체에서 사이드카 캡션 소스를 제거하는 유틸리티 메서드를 추가하는 클래스
  • 다음과 같은 문제가 수정되었습니다. video_duration URL 매개변수는 보고를 위해 Brightcove의 Analytics 백엔드에 대한 요청에 포함되지 않았습니다.

캐스트 플러그인 변경 사항:

  • Cast 수신기 앱의 사용자 지정 플레이어에 대한 URL 설정에 대한 지원이 추가되었습니다. 자세한 내용은 Android SDK 지원 문서를 참조하세요.

  • 선택한 캡션 및 오디오 트랙 정보를 Cast 수신기로 보내기 위한 지원이 추가되었습니다.

    알려진 비회귀 문제는 재생 중에 Cast 세션이 닫힌 경우 매니페스트 내 캡션 선택 항목이 발신자에게 자동으로 표시되지 않는 다음 릴리스에서 해결될 것입니다.

    이 문제에 대한 해결 방법은 전송 세션을 열기 전에 선택한 항목을 다시 선택하고 보낸 사람 장치에서 세션을 다시 열 때 표시할 수 있다는 것입니다.

오프라인 재생 변경 사항:

  • 속성 설정에 대한 지원이 추가되었습니다. 초록오프라인카탈로그수업. 오프라인 카탈로그 속성은 두 가지 방법으로 설정할 수 있습니다.

    • 로맵<문자열, 문자열>키/값 쌍의 setProperties(지도<문자열, 문자열>)방법
    • 통해 getProperties().put(키, 값)방법
  • 커스텀 생성 지원 추가알림 채널에서다운로드 관리자수업. 자세한 내용은 Android SDK 지원 문서를 참조하세요.

이전 릴리스

버전 6.16.2

2021년 5월 10일

SDK 변경 사항:

  • 텍스트가 종류와 함께 추적되는 문제를 수정했습니다. 자막구문 분석되거나 표시되지 않았습니다.
  • 텍스트 트랙에위치추가 속성줄 왼쪽또는줄 오른쪽속성이 구문 분석되거나 표시되지 않았습니다.

엑소플레이어 2 변경 사항:

  • 잘못된 색인을 사용하여 현재 선택한 오디오 언어를 가져올 수 있는 문제를 수정했습니다.

Omniture 플러그인 변경 사항:

  • 플레이어가 파괴된 후에도 심장 박동이 계속되는 문제를 수정했습니다.

SSAI 플러그인 변경 사항:

  • 중첩된 VAST 및 VMAP 확장 요소에 대한 구문 분석 지원 추가

    확장 요소는 중첩 깊이에 관계없이 SSAI 플러그인에 의해 실행되지 않으므로 이 지원은 모든 깊이의 중첩 확장 요소가 있는 VAST 및 VMAP 구문 분석이 실패하지 않도록 보장합니다.

  • VAST에 대한 구문 분석 지원 추가광고 인증집단

    광고 인증요소는 SSAI 플러그인에 의해 실행되지 않으므로 이 지원은광고 인증요소는 실패하지 않습니다.

    게다가, 광고 인증요소는 VAST 4.0 기능이며 Brightcove Android SDK는 아직 VAST 4.x를 완전히 지원하지 않습니다. 이 구문 분석 지원은 일반적으로 Android용 Brightcove 네이티브 SDK에서 VAST 4.x에 대한 지원으로 간주되어서는 안 됩니다.

버전 6.16.1

2021년 1월 12일

SDK 변경 사항:

  • SSAI VOD 비디오에 대한 개선된 비디오 참여 보고.
  • NS DashWidevineAuthorizer클래스는 더 이상 사용되지 않습니다. 이용해주세요 WidevineAuthorizer대신 수업.

IMA 플러그인 변경 사항:

  • Google IMA SDK가 버전 3.21.1로 업데이트되었습니다.
  • 건너뛸 수 있는 광고에 대한 지원이 향상되고 건너뛸 수 있는 광고가 제대로 종료되지 않고 콘텐츠 재생을 재개한 후 광고 오디오가 들릴 수 있는 문제가 수정되었습니다.
  • 사용자 제공 MIME 유형이 플러그인의 빌더에 의해 설정되지 않는 광고에 대한 MIME 유형 설정 문제가 수정되었습니다.
  • 다음을 위한 플러그인 빌더 편의 메소드가 추가되었습니다.
    • 광고 MIME 유형 설정 AdsRenderingSettings개체 - 참조 Builder.setMimeTypes(목록) : 무효의
    • 광고 동영상 로드 시간 초과 설정 AdsRenderingSettings개체 - 참조 Builder.setLoadVideoTimeout(int) : 무효
    • 디버그 모드 설정 ImaSdk설정개체 - 참조 Builder.setDebugMode(boolean) : 무효
  • 참고:

    이러한 편의 메서드는 멤버 메서드입니다. AdsRenderingSettings그리고 ImaSdk설정 Google IMA SDK 개체. 빌더를 통해 직접 호출되는 경우 이러한 메서드는 AdsRenderingSettings그리고 ImaSdk설정빌더에서 기본적으로 생성되는 객체.

    사용자 정의를 만들기로 결정한 경우 AdsRenderingSettings그리고 ImaSdk설정 IMA 플러그인에서 사용하기 위한 Google IMA SDK 개체, 이러한 개체를 호출할 수 있습니다.' 직접 방법:

    • AdsRenderingSettings.setMimeTypes(목록) : 무효의
    • AdsRenderingSettings.setLoadVideoTimeout(int) : 무효
    • ImaSdkSettings.setDebugMode(boolean) : 무효

SSAI 플러그인 변경 사항:

  • 원격 비디오 개체로 생성되는 라이브 SSAI 스트림에 대한 지원이 추가되었습니다.

버전 6.15.1

2020년 9월 22일

SDK 변경 사항:

  • Android용 Brightcove 기본 SDK는 이제 플레이어 컨트롤 및 UI 레이블에서 아랍어(RTL) 언어를 지원합니다.

  • 참고: 아랍어(RTL) 텍스트는 지원되지만 현재 LTR(왼쪽에서 오른쪽) 플레이어 컨트롤 레이아웃만 지원됩니다.

  • 특정 문제가 수정되었습니다. video_view그리고video_engagement Analytics 이벤트 매개변수에는 다음 매개변수에 대한 비서양 아라비아 숫자가 포함될 수 있습니다.

    • :video_view
      • start_time_ms
      • load_time_ms
    • :video_engagement
      • range
      • video_seconds_viewed

캐스트 플러그인 변경 사항:

  • 이제 Cast 플러그인은 캐스트 컨트롤 및 UI 레이블에서 아랍어(RTL) 텍스트를 지원합니다.

  • 아랍어 지원은 Cast 확장 컨트롤러의 디스플레이(비디오 경과 시간 및 지속 시간)로 확장되지 않습니다.

  • 이러한 구성요소의 표시는 Google Cast API에 의해 제어됩니다.

오프라인 재생 플러그인 변경 사항:

  • 오프라인 재생 플러그인은 이제 UI 레이블에서 아랍어(RTL) 텍스트를 지원합니다.

펄스 플러그인 변경 사항:

  • Pulse 플러그인은 이제 광고 컨트롤 및 UI 레이블에서 아랍어(RTL) 텍스트를 지원합니다.

SSAI 플러그인 변경 사항:

  • SSAI 플러그인은 이제 광고 컨트롤 및 UI 레이블에서 아랍어(RTL) 텍스트를 지원합니다.
