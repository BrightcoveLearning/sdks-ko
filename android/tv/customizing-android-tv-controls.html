소개

TV용 Brightcove 미디어 컨트롤러에는 되감기 버튼과 빨리 감기 버튼이 있습니다. 이 버튼 중 하나를 클릭하면 기본적으로 3초 뒤로 또는 앞으로 탐색합니다.

두 버튼 중 하나를 길게 누르면 탐색 위치가 40밀리초마다 콘텐츠 3초의 속도로 업데이트됩니다. 즉, 되감기 또는 빨리 감기 버튼을 1초마다 누르고 있으면 콘텐츠가 75초 앞으로 이동합니다.

탐색 버튼 구성

되감기 및 빨리 감기 버튼은 다음을 통해 구성할 수 있습니다. EventType.SEEK_CONTROLLER_CONFIGURATION이벤트.

다음 속성이 둘 다에 적용되므로 단순화를 위해 이 버튼 중 하나를 탐색 버튼이라고 합니다.

이벤트 속성 기본값 설명
Event.SEEK_DEFAULT 3000ms 탐색 버튼이 탐색할 기본 탐색 값(밀리초)입니다.
Event.SEEK_RELATIVE_ENABLED 거짓 활성화된 경우 탐색 값이EventType.VIDEO_DURATION_CHANGED이벤트. 백분율은 다음에 의해 설정됩니다. Event.SEEK_PERCENTAGE .
Event.SEEK_PERCENTAGE 1% 기본 탐색 값을 계산하는 데 사용되는 비디오 지속 시간에 대한 백분율입니다.
Event.SEEK_ON_HOLD_WAIT_TIME 500ms 키 이벤트가 일반 누름 대신 길게 누름으로 간주될 때까지 대기하는 시간(밀리초)입니다.
Event.SEEK_ON_HOLD_UPDATE_FREQ 40ms 전송에 사용된 시간(밀리초) EventType.SEEKBAR_DRAGGING_PROGRESS이벤트를 통해 미디어 컨트롤러 탐색 표시줄을 업데이트합니다.

이벤트 속성 사용자 정의

위의 표에 나열된 이벤트 속성에서 추가 이벤트 속성이 필요할 수 있습니다. 예상 값을 정의하고EventType.SEEK_CONTROLLER_CONFIGURATION이벤트.

이벤트에 사용자 정의 속성을 추가하려면 다음 단계를 따르세요.

  1. 사용자 정의 값을 정의하십시오. 

    private static final int DEFAULT_TV_SEEK_TIME = (int) TimeUnit.SECONDS.toMillis(3);
private static final int DEFAULT_TV_ON_HOLD_WAIT_TIME = (int) TimeUnit.SECONDS.toMillis(1);
private static final int DEFAULT_TV_SEEK_PERCENTAGE = 1;
private static final int DEFAULT_TV_ON_HOLD_UPDATE_FREQUENCY_TIME = 500;

  2. 속성 맵을 생성하는 메서드를 포함하고 원하는 값을 추가하고EventType.SEEK_CONTROLLER_CONFIGURATION이벤트. 

    private void setupTelevisionMode() {
   Map<String, Object> properties = new HashMap<>();
   properties.put(Event.SEEK_DEFAULT, DEFAULT_TV_SEEK_TIME);
   properties.put(Event.SEEK_RELATIVE_ENABLED, true);
   properties.put(Event.SEEK_PERCENTAGE, DEFAULT_TV_SEEK_PERCENTAGE);
   properties.put(Event.SEEK_ON_HOLD_WAIT_TIME, DEFAULT_TV_ON_HOLD_WAIT_TIME);
   properties.put(Event.SEEK_ON_HOLD_UPDATE_FREQ, DEFAULT_TV_ON_HOLD_UPDATE_FREQUENCY_TIME);

   eventEmitter.emit(EventType.SEEK_CONTROLLER_CONFIGURATION, properties);
}

  3. 탐색 컨트롤러 구성을 설정하려면 앱이 TV 모드인지 확인하세요. 

    if (BrightcoveMediaController.checkTvMode(mBaseVideoView.getContext())) {
   setupTelevisionMode();
}
