개요

Android 6.15.0 이상용 Brightcove 기본 SDK는 Google IMA SDK 버전 3.19.4를 지원합니다. IMA 광고를 사용하는 경우 이 버전의 IMA 플러그인을 사용하기 위해 알아야 할 몇 가지 사항이 있습니다. 여기에는 다음이 포함됩니다.

  • Google IMA SDK의 변경사항
  • Brightcove IMA 플러그인 변경 사항
  • IMA 플러그인 6.15.0으로 마이그레이션</a

Google IMA SDK의 변경사항

Brightcove Native SDK는 Google IMA SDK를 버전 3.11.2에서 버전 3.19.4로 업데이트했습니다. 가장 관련성이 높은 변경 사항은 다음과 같습니다.

  • SdkFactory.createAdDisplayContainer()더 이상 사용되지 않습니다
  • AdDisplayContainer.setPlayer(VideoAdPlayer)더 이상 사용되지 않습니다
  • AdDisplayContainer.setAdContainer(ViewGroup)더 이상 사용되지 않습니다
  • AdsRequest.setAdDisplayContainer(AdDisplayContainer)제거된다

전체 릴리스는 다음을 참조하십시오. Google IMA Android SDK 출시 내역문서.

Brightcove IMA 플러그인 변경 사항

Brightcove IMA 플러그인은 새로운 Google IMA SDK 버전과 완벽하게 호환되도록 몇 가지를 변경했습니다. 여기에는 다음이 포함됩니다.

  • GoogleIMAComponent 빌더 사용
  • AdDisplayContainer 만들기
  • GoogleIMAVideoAdPlayer에서 더 이상 사용되지 않는 메소드

GoogleIMAComponent 빌더 사용

Builder를 사용하여 생성하는 것이 좋습니다. GoogleIMAComponent사례. 이렇게 하면 더 쉽게 구성할 수 있습니다. GoogleIMAComponent .

googleIMAComponent = new GoogleIMAComponent.Builder(brightcoveVideoView, eventEmitter)
  .setUseAdRules(true)
  .setImaSdkSettings(customIMASDKSettings)
  .setAdsRenderingSettings(customAdsRenderingSettings)
  .setAdDisplayContainerFactory(customAdDisplayContainerFactory)
  .build();
}

AdDisplayContainer 만들기

Google IMA SDK가 제거되었기 때문에AdsRequest.setAdDisplayContainer(...) , 이제 Brightcove IMA 플러그인이 다음을 사용하여 자동으로 생성합니다. GoogleIMAVideoAdPlayer그리고BaseVideoView .

플러그인이 필요합니다AdDisplayContainerFactory만들기AdDisplayContainer .

public interface AdDisplayContainerFactory {
  /**
  * Creates the AdDisplayContainer with the provided GoogleIMAVideoAdPlayer
  * and the ViewGroup retrieved with getViewContainer()
  *
  * @param googleIMAVideoAdPlayer the Brightcove Ad Player for Google IMA
  */
  AdDisplayContainer createAdDisplayContainer(GoogleIMAVideoAdPlayer googleIMAVideoAdPlayer);

  /**
  * Returns the ViewGroup container used for both,
  * the creation of the GoogleIMAVideoAdPlayer and the AdDisplayContainer
  */
  ViewGroup getViewContainer();
}

내부적으로 플러그인은 다음을 사용합니다. DefaultAdDisplayContainerFactory , NS AdDisplayContainerFactory , 생성AdDisplayContainer . NS getViewContainer()를 반환BaseVideoView생성자에 전달되고createAdDisplayContainer메서드는 다음을 반환합니다.

ImaSdkFactory.createAdDisplayContainer(getViewContainer(), googleIMAVideoAdPlayer);

GoogleIMAVideoAdPlayer에서 더 이상 사용되지 않는 메소드

IMA 플러그인 6.15.0으로 마이그레이션

Brightcove IMA 플러그인 6.15.0으로 마이그레이션하려면 다음 단계를 따르세요.

  • 종속성 버전 업데이트
  • 더 이상 사용되지 않는 메소드 제거

종속성 버전 업데이트

다음 종속성 버전을 사용하도록 앱을 업데이트합니다.

  • 모든 Brightcove SDK 종속성이 버전 6.15.0으로 설정됨
  • Google IMA SDK 버전 3.19.4 사용
  • Google Play 서비스 버전 19.2.0 사용

다음은 의 예입니다. build.gradle파일:

//build.gradle
dependencies {
 //Brightcove SDK dependencies
 implementation "com.brightcove.player:android-sdk6.15.0:"
 implementation "com.brightcove.player:exoplayer2:6.15.0"
 implementation "com.brightcove.player:android-ima-plugin:6.15.0"

 //Google IMA SDK
 implementation "com.google.ads.interactivemedia.v3:interactivemedia:3.19.4"

 //Google Play Services
 implementation "com.google.android.gms:play-services-ads:19.2.0"
 ...
}

더 이상 사용되지 않는 메소드 제거

현재 Google IMA 설정은 다음과 유사할 수 있습니다.

AdDisplayContainer container = sdkFactory.createAdDisplayContainer();
container.setPlayer(googleIMAComponent.getVideoAdPlayer());
container.setAdContainer(brightcoveVideoView);

이 세 가지 방법은 더 이상 사용되지 않으며 사라질 것입니다.

당신은 지금 전화해야합니다googleIMAComponent.getAdDisplayContainer()그리고 그것은 돌아올 것이다AdDisplayContainer와 관련된GoogleIMAVideoAdPlayer , 또는 사용할 수 없는 경우 null입니다.

생성에 대한 자세한 내용은AdDisplayContainer , 참조 AdDisplayContainer 만들기부분.

다음 메소드도 Google IMA SDK에서 제거되었습니다.

adsRequest.setAdDisplayContainer(container);

대신 다음을 사용하십시오. AdDisplayContainer Google IMA의 인스턴스를 만들려면AdsLoader .

더 이상 사용되지 않는 메서드 및 대체 방법에 대한 자세한 내용은 다음을 참조하세요. GoogleIMAVideoAdPlayer에서 더 이상 사용되지 않는 메소드부분.
