개요

Native Player SDK는 Brightcove Web Player와 유사한 플레이어 컨트롤을 제공하여 플랫폼 전반에 걸쳐 일관된 사용자 경험을 제공합니다. 기본 제공되는 Brightcove 플레이어 컨트롤을 사용하거나 고유한 색상, 스타일 및 버튼으로 사용자 지정할 수 있습니다.

노트:

이러한 기능에 대해 자세히 알아보려면브라이트코브 네이티브 플레이어 컨트롤문서.

Android용 네이티브 SDK의 전체 속성 값 목록은 SDK 속성 값 찾아보기이 문서의 섹션.

플레이어 컨트롤 사용자 지정

Brightcove 플레이어 컨트롤은 캡션, 비디오 되감기 및 전체 화면 모드를 지원합니다. 컨트롤을 표시하거나 숨기기 위한 애니메이션 스타일을 선택할 수도 있습니다.

컨트롤 색상 변경

쉽게 사용자 정의할 수 있는 Brightcove 플레이어 컨트롤 색상을 회사 브랜딩 자료와 일치하도록 변경할 수 있습니다.

  1. 귀하의메인 액티비티.자바위와 같이 파일을
  2. 당신의 src/main/res/values/colors.xml또는 res/values/strings.xml파일에 다음 코드를 포함합니다. 
    <?xml version="1.0" encoding="utf-8"?>
<resources>

    <!-- Define the new colors. -->
    <color name="bc_orange">#F3951D</color>
    <color name="magenta_semi_trans">#4DFF00FF</color>

    <!-- Change the progress colors to magenta/black/orange, all transparent. -->
    <color name="bmc_seekbar_played">@color/magenta_semi_trans</color>
    <color name="bmc_seekbar_buffered">@color/black_semi_trans</color>
    <color name="bmc_seekbar_track">@color/bc_orange</color>

</resources>

플레이어 컨트롤 표시/숨기기

비디오 재생 중에 플레이어 컨트롤을 표시하거나 숨기는 세 가지 방법이 있습니다.

  1. 사용자는 장치 화면을 탭하여 컨트롤 표시와 숨기기 사이에서 플레이어 컨트롤을 전환할 수 있습니다.

  2. 당신은 사용할 수 있습니다BrightcoveMediaController수업. 다음은 정의된 플레이헤드 위치 범위에 대한 플레이어 컨트롤을 표시하는 방법의 한 예입니다.

    brightcoveVideoView.getEventEmitter().on(EventType.PROGRESS, new EventListener() {
    @Override
    public void processEvent(Event event) {
        int playheadPosition = brightcoveVideoView.getCurrentPosition();
        if(playheadPosition > 24000 && playheadPosition < 29000) {
            BrightcoveMediaController controller = brightcoveVideoView.getBrightcoveMediaController();
            controller.show();
        }
    }
});

    호출하려고하면 null 포인터 예외가 발생합니다. getBrightcoveMediaController()동영상이 로드되기 전에 당신은 또한 들을 수 있습니다DID_SET_VIDEO이 메서드를 호출하기 전에 이벤트.

    이 기본 동작을 재정의하지 않는 한 플레이어 컨트롤은 표시한 후 3초 후에 자동으로 숨겨집니다.

  3. 다음과 같이 이벤트를 사용할 수 있습니다.

    import static com.brightcove.player.mediacontroller.ShowHideController.SHOW_MEDIA_CONTROLS;
import static com.brightcove.player.mediacontroller.ShowHideController.HIDE_MEDIA_CONTROLS;
...
eventEmitter.emit(HIDE_MEDIA_CONTROLS);
eventEmitter.emit(SHOW_MEDIA_CONTROLS);

플레이어 컨트롤 비활성화

앱 레이아웃과 기능을 디자인하는 동안 플레이어 컨트롤을 꺼야 할 수도 있습니다. 이것은 설정하여 수행 할 수 있습니다MediaController다음과 같이 null을 지정합니다.

@
Override
public View onCreateView(LayoutInflater inflater, ViewGroup container, Bundle savedInstanceState) {
   View rootView = inflater.inflate(R.layout.fragment_player, container, false);
   brightcoveVideoView = (BrightcoveExoPlayerVideoView) rootView.findViewById(R.id.brightcove_video_view);
   super.onCreateView(inflater, container, savedInstanceState); // Need to be called after brightcoveVideoView is set.
   brightcoveVideoView.setMediaController((MediaController) null); // Need to be called after onCreateView.
   return rootView;
}

이 예에서는 다음을 사용합니다. BrightcovePlayerFragment .

NS BrightcoveMediaController클래스는 기본 Brightcove 플레이어 컨트롤을 제공하며 컨트롤 막대, 탐색 막대 및 미디어 컨트롤 버튼 세트를 포함하여 이를 관리하고 사용자 정의할 수 있습니다. Brightcove Player 컨트롤을 비활성화하여 Android 기본 컨트롤을 사용할 수도 있습니다.

전체 화면 모드를 수동으로 제어

일반적으로 사용자는 제어 막대에서 전체 화면 버튼을 눌러 전체 화면 보기를 제어합니다. 코드에서 이 기능을 수동으로 제어하려는 경우가 있습니다.

전체 화면 모드를 설정하려면 다음 코드를 사용하십시오.

brightcoveVideoView.getEventEmitter().emit(EventType.ENTER_FULL_SCREEN);

전체 화면 모드를 종료하려면 다음 코드를 사용하십시오.

brightcoveVideoView.getEventEmitter().emit(EventType.EXIT_FULL_SCREEN);

이벤트 유형의 전체 목록은 Android 참조용 플레이어 SDK로 이동합니다. 이벤트 유형수업 문서.

또는 다음을 사용할 수 있습니다. fullScreen()그리고normalScreen()행동 양식. 이러한 방법은 다음에서 찾을 수 있습니다. Android 참조용 플레이어 SDK , 브라이트코브플레이어수업 문서.

SDK 속성 값 찾아보기

배열, 색상, 차원, ID, 정수, 문자열 및 스타일을 포함한 전체 값 목록을 보려면 다음을 수행하십시오.

  1. 찾기 android-native-player-public-xxxzip파일을 압축 해제합니다.

  2. 에서 SDK폴더를 열고 android-sdk-xxxaar파일.

    여는 한 가지 방법아아파일은 확장자를 변경하는 것입니다아아에게지퍼압축을 풉니다.
  3. 열기 res/values/values.xml파일. 여기에서 애플리케이션에서 참조할 수 있는 값 목록을 찾을 수 있습니다.
