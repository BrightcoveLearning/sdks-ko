소개

Brightcove 카탈로그에서 비디오를 검색할 때 SDK는 재생할 소스를 결정해야 합니다. 이것은기본 소스 선택 컨트롤러그리고 ExoPlayerSourceSelectionController구성 요소.

기본 선택 프로세스

Android용 Brightcove Player SDK는 기본 소스 선택 프로세스를 사용합니다. 단순화하기 위해 Brightcove Playback API 및 Google의 ExoPlayer에 대한 SDK 지원에 중점을 둘 것입니다.

기본 선택 프로세스는 다음과 같이 정의됩니다.

  1. Android 6.4 이상용 Brightcove 기본 SDK를 사용하면 사용 가능한 경우 모든 소스 유형에 대한 HTTPS 전달이 선택됩니다.

    둘 다기본 소스 선택 컨트롤러그리고 ExoPlayerSourceSelectionController HTTPS 버전 선택 시도deliveryType선택된. HTTPS 소스가 없으면 선택 컨트롤러가 사용 가능한 첫 번째 소스를 선택합니다.

    비디오에 WebVTT 캡션/자막에 대한 HTTP 및 HTTPS 소스가 모두 있는 경우 Android용 기본 SDK는 HTTPS 버전을 반환하지 않을 수 있습니다. Brightcove는 이러한 상황을 인지하고 있으며 향후 릴리스에서 수정할 예정입니다.
  2. NS BrightcoveExoPlayerVideoView가 있는 첫 번째 소스를 찾습니다. deliveryType NS MPEG-DASH , 그리고 둘 다 아닌 프로필항아리:hbbtv ...도 아니다항아리:dvb . SDK는 이러한 DASH 프로필로 재생을 지원하지 않으므로 소스에서 필터링됩니다.
  3. 위의 항목을 찾을 수 없는 경우 소스 선택은deliveryType NS HLS .
  4. 그러한 소스가 발견되지 않으면 다음을 사용하여 소스로 폴백합니다. deliveryType NS MP4 256kBps에 가장 가까운 비트 전송률.

소스를 찾으면 구현이 다음과 같이 계속됩니다.

  1. 발견된 경우 선택한 소스가 에 대한 응답으로 내보내집니다 SELECT_SOURCE이벤트.
  2. 기본적으로비디오 재생 컨트롤러핸들 SELECT_SOURCE응답, SET_SOURCE이벤트.
  3. NS VideoDisplayComponent또는 하위 클래스가 SET_SOURCE URL을 기본 플레이어(ExoPlayer 또는 MediaPlayer)에 로드하여 이벤트를 생성합니다.
  4. 재생이 시작되면 기본 플레이어는 HLS 및 DASH 콘텐츠로 비트 전송률을 조정해야 합니다. HLS를 사용하면 기본 플레이어가 마스터 매니페스트에 나열된 첫 번째 비트 전송률로 시작합니다.

기본 코드 변환 설정을 사용할 때 첫 번째 HLS 소스는 모든 개별 소스(변환)를 가리키는 마스터 m3u8 재생 목록이어야 합니다.

HLS 소스 설정

SDK의 기본 소스 선택 정책이 호출되기 전에 원하지 않는 전달 유형을 필터링하기 위해 앱 수준에서 소스 선택을 사용자 정의할 수 있습니다. 이렇게 하려면 Playback API의 JSON 응답에서 Video 객체로 설정된 소스 컬렉션을 수정합니다.

이 예에서는 HLS 소스만 사용합니다.

catalog.findVideoByReferenceID(videoReferenceId, new com.brightcove.player.edge.VideoListener() {
	@Override
	public void onVideo(Video video) {
	 sourceSelectionFilter(DeliveryType.HLS, video);
	 brightcoveVideoView.add(video);
	 brightcoveVideoView.start();
	}
});

그리고sourceSelectionFilter()방법은 다음과 같습니다.

public static void sourceSelectionFilter(@NonNull DeliveryType deliveryTypeFilter, @NonNull Video video) {
 if (video.getSourceCollections().containsKey(deliveryTypeFilter)) {
   // Only remove the other delivery types if the desired type is in our Source collections
   for (DeliveryType deliveryType : DeliveryType.values()) {
     if (!deliveryType.equals(deliveryTypeFilter)) {
            video.getSourceCollections().remove(deliveryType);
     }
   }
 }
}
