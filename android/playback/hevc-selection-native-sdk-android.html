개요

H.265라고도 하는 HEVC(고효율 비디오 코딩)는 비디오 파일 크기를 줄여 고품질 재생을 가능하게 하는 비디오 압축 표준입니다.

Android용 Brightcove 기본 SDK는 다음 HEVC 선택 프로세스를 사용합니다.

  1. HEVC 소스 선택
  2. HEVC 변환 선택

노트:

  • Android 6.13.1 이상용 Brightcove 네이티브 SDK에서 지원
  • API 레벨 21(Lollipop 버전 5.0) 이상을 사용하는 Android 기기에서 작동

HEVC 소스에는 문자열이 포함되어 있습니다. hvc URL에서.

Android 소스 선택기용 기본 SDK는 HEVC 소스를 선호합니다. 위의 예에서는 HEVC HTTPS 소스를 선택합니다.

네이티브 SDK는 에 정의된 기본 선택 프로세스를 계속합니다. 기계적 인조 인간: 소스 선택문서:

  1. HTTP보다 HTTPS 선호
  2. HLS보다 DASH를 선호
  3. 재생을 위해 HLS를 선택하는 경우 사용 가능한 HLS의 가장 높은 버전을 선호하십시오.

HEVC 변환 선택

수집 구성에 따라 HEVC 소스가 다른 변환에 포함될 수 있습니다. 다음이 있을 수 있습니다.

  • HEVC 변환만
  • HEVC 및 기타 코덱 혼합
이러한 종류의 선택은 ExoPlayer를 사용할 때만 적용됩니다.

HEVC 변환만

비디오 소스의 변환에 HEVC 코덱만 있는 경우 변환 선택이 필요하지 않습니다. ExoPlayer는 장치 기능과 사용 가능한 대역폭에 따라 적절한 변환을 선택합니다.

HEVC 및 기타 코덱 혼합

비디오 소스에 HEVC와 다른 코덱(예: AVC)이 혼합된 변환이 있는 경우 변환 선택 프로세스가 호출됩니다. 네이티브 SDK는 소스를 ExoPlayer에 로드할 때 다음을 수행합니다. 선택 재정의비디오 변환에서 다른 코덱보다 HEVC를 선호합니다.

장치에서 지원하는 변환 형식의 비디오가 있는 경우 변환 프로세스는 다음과 같습니다.

  1. 기본 SDK는 모든 HEVC 변환을 선택합니다.
  2. 동일한 해상도로 기존 HEVC를 복제하는 비 HEVC 변환을 무시합니다.

이것은 ExoPlayer가 대부분의 경우 HEVC를 선호하도록 합니다. 다음은 두 가지 가능한 시나리오입니다.

  • 코덱 믹스 복제
  • 반반

코덱 믹스 복제

이 시나리오에서는 대부분의 HEVC 및 AVC 코덱의 해상도가 복제됩니다. 예를 들면 다음과 같습니다.

AVC <Representation bandwidth="1989000" height="720" width="1280">

AVC <Representation bandwidth="3761000" height="720" width="1280">
HVC <Representation bandwidth="304000" height="270" width="480">
HVC <Representation bandwidth="548000" height="360" width="640">

위의 비디오 변환이 주어지면 Brightcove Natvie SDK는 다음 변환을 선택합니다. 

AVC <Representation bandwidth="1202000" height="540" width="960">
HVC <Representation bandwidth="304000" height="270" width="480">
HVC <Representation bandwidth="548000" height="360" width="640">
HVC <Representation bandwidth="993000" height="432" width="768">
HVC <Representation bandwidth="1301000" height="576" width="1024">
HVC <Representation bandwidth="1605000" height="720" width="1280">
HVC <Representation bandwidth="2809000" height="720" width="1280">

해상도가 기존 HEVC 변환을 복제하지 않기 때문에 하나의 AVC 변환만 선택되었습니다.

반반

이 시나리오에는 AVC 코덱을 사용한 저해상도 변환과 HVC 코덱을 사용한 고해상도 변환이 있습니다. 예를 들면 다음과 같습니다. 

AVC <Representation bandwidth="453000" height="270" width="480">
AVC <Representation bandwidth="704000" height="360" width="640">
AVC <Representation bandwidth="903000" height="360" width="640">
AVC <Representation bandwidth="1202000" height="540" width="960">
AVC <Representation bandwidth="3761000" height="720" width="1280">
HVC <Representation bandwidth="1301000" height="576" width="1024">
HVC <Representation bandwidth="1605000" height="720" width="1280">
HVC <Representation bandwidth="2809000" height="720" width="1280">

이 경우 HEVC 변환을 복제하는 해상도의 AVC 코덱이 없기 때문에 기본 SDK는 모든 변환을 선택하여 ExoPlayer가 사용 가능한 대역폭을 기반으로 변환을 선택할 수 있도록 합니다.

기본 SDK는 이 시나리오의 모든 경우에 HEVC 선택을 강제하지 않으므로 가장 낮은 변환을 높은 비트 전송률 HEVC 변환으로 제한하지 않습니다. 이로 인해 네트워크 연결이 좋지 않아 사용자 환경이 좋지 않을 수 있습니다.
