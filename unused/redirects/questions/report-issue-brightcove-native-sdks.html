개요

Brightcove Native Player SDK로 개발하는 동안 문제가 발생했거나 예기치 않게 발생했습니까? 원래대로 돌아갈 수 있도록 무슨 일이 있었는지 알려주세요.

보고 프로세스를 신속하게 처리하려면 버그를 제출하기 전에 다음을 수행하십시오.

  1. 이것이 알려진 문제인지 확인하십시오.
  2. 문제를 Brightcove SDK로 격리

문제가 알려진 문제가 아니고 Brightcove SDK에 격리된 것으로 확인되면 버그를 제출할 준비가 된 것입니다. 다음 단계를 따르세요.

  1. 사례별 정보 제공
  2. 도움이 되는 추가 정보 포함
  3. 버그 제출

알려진 문제인지 확인

먼저 개발자 포럼에서 문제가보고되었는지 확인하세요.

  • Brightcove Player SDK 개발자 포럼

그런 다음 오류가 개발자 문서에 명시된 알려진 문제인지 확인합니다.

  • 기본 SDK 알려진 문제

문제 분리

제공된 샘플 응용 프로그램 중 하나에서 문제를 재현해 보십시오. 이러한 앱은 기본 구현을 제공하며 문제가 SDK에 있는지 아니면 외부 요인에 있는지 밝히는 데 도움이 될 수 있습니다.

다음 샘플 앱 중 하나로 테스트합니다.

  • iOS 샘플 앱용 Brightcove SDK
  • Android 샘플 앱용 Brightcove SDK

정보를 제공하다

사례별 정보를 제공하려면 다음을 수집하십시오.

  • 발생한 오류 동작/예상 동작
  • 기기별 세부정보(제조업체, 모델 버전)
  • 사용된 SDK 버전
  • 충돌 로그(Android용 logcat, 콘솔/장치 충돌 로그)
  • 재현 단계

추가 정보

가능한 경우 다음 세부정보를 포함합니다.

  • 코드 스니펫
  • 오류를 보여주는 샘플 앱
  • 문제를 해결하기 위해 시도한 것
