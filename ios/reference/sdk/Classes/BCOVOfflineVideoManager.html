기본 비디오 외에 다운로드할 AVMediaSelections의 배열입니다. BCOVVideo에 사용 가능한 AVMediaSelections는 -urlAssetForVideo:error: 다음에 [AVAsset preferredMediaSelection] 또는 [AVAsset allMediaSelections]를 호출하여 검색할 수 있습니다. mediaSelections에 nil을 전달하면 선호하는 AVMediaSelections를 가져옵니다. iOS 13 이상에서는 requestVideoDownload:mediaSelections:parameters:completion:에 대한 단일 호출을 사용하여 모든 비디오 자산을 다운로드해야 합니다. 추가 트랙은 나중에 다운로드할 수 없습니다.
매개변수

이 다운로드 요청에 사용된 매개변수의 NSDictionary입니다. 0일 수 있습니다. 유효한 매개변수는 kBCOVOfflineVideoManagerRequestedBitrateKey, kBCOVFairPlayLicensePurchaseKey, kBCOVFairPlayLicenseRentalDurationKey입니다.
완료 핸들러

요청이 성공적으로 이루어진 후 새 offlineVideoToken을 수신하는 블록입니다. 오류가 있는 경우 offlineVideoToken은 nil이 되고 error는 NSError를 보고합니다.

논의

Video Cloud Dynamic Delivery와 함께 FairPlay 비디오를 사용하는 경우 FairPlay 애플리케이션 인증서가 자동으로 다운로드되어 필요에 따라 사용됩니다. 레거시 Video Cloud FairPlay 비디오를 사용하는 경우 -[addFairPlayApplicationCertificate:identifier:]를 사용하여 다운로드를 시작하기 전에 BCOVOfflineVideoManger에 필요한 애플리케이션 인증서를 추가해야 합니다.

다운로드한 비디오를 식별하는 데 사용되는 오프라인 비디오 토큰입니다.

논의

반환 BCOV비디오또한 다운로드한 썸네일 및 포스터 이미지 파일 경로, FairPlay 라이선스 만료 날짜, 다운로드 시작 및 종료 시간 등과 같이 위에 나열된 모든 새 속성이 포함되어 있습니다.

비디오의 downloadState가 BCOVOfflineVideoDownloadStateCompleted그러면 토큰과 연결된 비디오가 캐시됩니다. 어떤 이유로든 캐시를 지워야 하는 경우 다음을 호출할 수 있습니다. clearOfflineVideoCache ~에 BCOVOfflineVideoManager .

선언

오프라인 캐시 지우기 BCOV비디오사물

- (무효)clearOfflineVideoCache

논의

언제 videoObjectFromOfflineVideoToken: downloadState가 인 비디오와 함께 사용됩니다. BCOVOfflineVideoDownloadStateCompleted그러면 BCOVOfflienVideoManager가 해당 개체를 캐시하여 성능을 향상시킵니다. 이 방법은 캐시를 정리하여 다음에 videoObjectFromOfflineVideoToken:신품이라고 합니다 BCOV비디오객체가 생성됩니다.

매개변수

이 다운로드 요청에 사용된 매개변수의 NSDictionary입니다. 유효한 매개변수는 kBCOVFairPlayLicensePurchaseKey, kBCOVFairPlayLicenseRentalDurationKey입니다.
완료 핸들러

요청이 성공적으로 이루어진 후 새 offlineVideoToken을 수신하는 블록입니다. 오류가 있는 경우 offlineVideoToken은 nil이 되고 error는 NSError를 보고합니다.

논의

비디오에 FairPlay 소스가 없으면 오류가 반환됩니다. FairPlay로 보호된 여러 비디오를 다운로드할 때 다운로드를 시작하기 전에 각 비디오에 대해 이 메서드를 호출하여 각 비디오에 대한 라이선스를 획득해야 합니다. 이렇게 하면 백그라운드에서 다운로드가 시작된 경우에도 비디오가 제대로 저장될 수 있습니다.

완료 핸들러

라이선스 갱신 요청이 완료된 후 호출되는 블록입니다. 요청 결과 오류가 발생하면 여기에 NSError로 반환됩니다. offlineVideoToken 매개변수는 첫 번째 매개변수로 전달된 것과 동일한 오프라인 비디오 토큰이 됩니다.

논의

최초 라이선스를 취득한 후 언제든지 새로운 FairPlay 라이선스를 요청할 수 있습니다.

매개변수에서 새 대여 기간을 지정하거나 대여 라이센스를 구매 라이센스로 변환할 수 있습니다.

새로운 FairPlay 라이선스를 사용하기 위해 비디오를 다시 다운로드할 필요는 없지만 Playback API에서 새 비디오 개체를 검색해야 합니다(직접 또는 BCOV재생 서비스수업). 참조된 비디오를 더 이상 Playback API를 통해 사용할 수 없는 경우 실질적으로 유사한 비디오를 사용할 수 있습니다. 즉, 비디오는 동일한 계정에 있어야 하고 동일한 FairPlay 구성을 가져야 합니다.

매개변수에 비트 전송률을 지정하면 안 됩니다. 비트 전송률을 변경하려면 현재 다운로드를 삭제한 다음 새 비트 전송률에서 완전히 새로운 다운로드를 시작해야 합니다.

이 방법은 오프라인 비디오 관리자에 FairPlay 애플리케이션 인증서를 추가하는 데 사용됩니다. ’사용 가능한 FairPlay 애플리케이션 인증서 목록.

- (void)addFairPlayApplicationCertificate:(NSData *)응용 프로그램 인증서 데이터 식별자:(NSString *)식별자

매개변수

응용 프로그램 인증서 데이터

NSData 개체의 FairPlay 애플리케이션 인증서. nil로 설정하면 지정된 식별자에 대한 기존 애플리케이션 인증서가 제거됩니다.
식별자

애플리케이션 인증서를 찾는 데 사용되는 문자열입니다.

이 문자열은 nil이 아닐 수 있습니다.

Dynamic Delivery의 경우 식별자는 이 인증서를 검색하는 데 사용된 URL이어야 합니다.

레거시 Video Cloud 계정의 경우 모든 계정에 적용하려면 kBCOVDefaultFairPlayApplicationCertificateIdentifier로 설정해야 합니다.

여러 기존 Video Cloud 계정을 사용하는 경우 이 매개변수를 계정 ID로 설정하십시오.

논의

인증서는 응용 프로그램의 수명 동안 유지됩니다. 인증서를 제거하려면 인증서 데이터와 동일한 식별자를 nil로 추가하십시오.

Dynamic Delivery를 사용하는 경우 응용 프로그램 인증서는 FairPlay 플러그인에 의해 자동으로 검색되므로 이 방법이 필요하지 않습니다. 그러나 이 방법을 사용하여 애플리케이션 인증서를 미리 로드하여 첫 번째 FairPlay로 암호화된 비디오의 재생 속도를 높일 수 있습니다.

인증서는 저장되고 후속 비디오에 재사용됩니다.
