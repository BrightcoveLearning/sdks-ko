BCOVFairPlay 참조

iOS용 Brightcove Player SDK용 FairPlay 플러그인, 버전 1.1.3.89

지원되는 플랫폼

iOS 8.0 이상.

tvOS 9.0 이상.

설치

Brightcove Player SDK용 FairPlay 플러그인은 iOS용 두 가지 설치 패키지인 정적 라이브러리 프레임워크와 동적 프레임워크를 제공합니다. 배포는 iOS 8.0 이상에서 지원됩니다.

FairPlay 플러그인은 tvOS 9.0 이상을 지원하는 동적 프레임워크를 제공합니다.

다음 iOS/tvOS + FairPlay 제한 사항에 유의하십시오.

  • 플러그인에 시뮬레이터 아키텍처가 포함되어 있지만 시뮬레이터에서 FairPlay 보호 콘텐츠를 재생할 수 없습니다.

  • FairPlay 스트리밍 콘텐츠는 AirPlay 미러링 모드에서 렌더링되지 않습니다. 자세한 내용은 2015 WWDC 프레젠테이션 502를 참조하세요. HLS를 위한 콘텐츠 보호 .

코코아팟

당신이 사용할 수있는코코아팟프로젝트에 Brightcove Player SDK를 추가합니다. 최신 정보를 찾을 수 있습니다 Brightcove-Player-SDK-FairPlay포드 스펙여기 . podspec은 iOS와 tvOS를 모두 지원합니다. CocoaPods 버전 1.0 이상이 필요합니다.

기본 포드 지정 Brightcove-Player-SDK-FairPlay정적 라이브러리 프레임워크를 설치합니다. 동적 프레임워크를 설치하려면 다음을 사용하여 포드를 선언합니다. 동적하위 사양: Brightcove-Player-SDK-FairPlay/다이내믹

정적 프레임워크 예:

포드 'Brightcove-Player-SDK-FairPlay'

동적 프레임워크 예:

포드 'Brightcove-Player-SDK-FairPlay/다이내믹'

항상 최신 버전의 Brightcove 소프트웨어를 사용하려면 최신 마스터 podspec 리포지토리를 유지 관리해야 합니다. CocoaPods 1.0.0부터 podspec 리포지토리 업데이트는 더 이상 자동 기능이 아니므로 마스터 리포지토리를 업데이트하려면 명령줄에서 다음을 실행하세요.

포드 리포지토리 업데이트

설명서

Brightcove Player SDK용 FairPlay 플러그인을 프로젝트에 수동으로 추가하려면:

  1. 의 최신 버전을 설치하십시오. 브라이트코브 플레이어 SDK .
  2. 플러그인의 최신 압축 릴리스를 다운로드하십시오. 릴리스 페이지 .
  3. 추가 BrightcoveFairPlay.framework당신의 프로젝트에.
  4. 에“빌드 설정”애플리케이션 대상의 탭에서“프레임워크 검색 경로”프레임워크에 대한 경로를 포함합니다. 프레임워크가 프로젝트와 다른 루트 디렉터리에 저장되지 않는 한 이 작업은 자동으로 수행되어야 합니다.
  5. 에“일반적인”응용 프로그램 대상의 탭에서 다음을 추가하십시오. “라이브러리와 바이너리 연결”부분:
    • BrightcoveFairPlay.framework
  6. (동적 프레임워크만 해당) “일반적인”응용 프로그램 대상의 탭에 추가‘ BrightcoveFairPlay.framework ’ ~로“임베디드 바이너리”부분.
  7. (동적 프레임워크만 해당) “빌드 단계”탭, 추가“스크립트 실행”명령으로 단계배시 $ {BUILT_PRODUCTS_DIR} /$ {FRAMEWORKS_FOLDER_PATH} /BrightcoveFairPlay.framework/strip-frameworks.sh . 확인하다“설치 시에만 스크립트 실행” . 이렇게 하면 앱 스토어 제출에 중요한 불필요한 아키텍처가 빌드에서 제거됩니다.
  8. (정적 프레임워크만 해당) “빌드 설정”응용 프로그램 대상의 탭에 추가 -오브제이C ~로“기타 링커 플래그”빌드 설정.

수입품

iOS용 Brightcove Player SDK는 몇 가지 다른 방법으로 코드로 가져올 수 있습니다. @import BrightcoveFairPlay; , #import <BrightcoveFairPlay/BrightcoveFairPlay.h>또는 #import <BrightcoveFairPlay/[특정 클래스].h> .

빠른 시작

Brightcove FairPlay는 Apple FairPlay와 iOS용 브라이트코브 플레이어 SDK . 다음 섹션에서는 기본 재생을 위해 코드를 설정하는 방법을 보여줍니다.

비디오 클라우드 동적 전달

Video Cloud Dynamic Delivery를 사용하면 FairPlay 애플리케이션 인증서에 대한 참조가 재생 서비스에서 반환된 BCOVVideo 개체에 포함됩니다. SDK는 이 참조를 사용하여 FairPlay 비디오를 재생할 때 애플리케이션 인증서를 검색합니다.

따라야 할 기본 단계는 다음과 같습니다.

@property(비원자, 강력) id<BCOVPlaybackController> playbackController;
---
-
---
  // // FairPlay 자격 증명을 Brightcove에 등록하는 데 사용되는 // 애플리케이션 ID 및 게시자 ID를 사용하여 BCOVFPSBrightcoveAuthProxy 개체를 생성합니다. BCOVFPSBrightcoveAuthProxy *proxy = [[BCOVFPSBrightcoveAuthProxy 할당] initWithPublisherId:<pub-id> applicationId:<app-id>]; BCOVPlayerSDKManager *sdkManager = [BCOVPlayerSDKManager 공유 관리자]; // Brightcove FairPlay는 BCOVPlaybackManager에 몇 가지 범주 메서드를 추가합니다. // 자체 FairPlay 라이선스 서버를 사용하는 경우 // 여기에서 자체 프록시 구현을 전달합니다. id<BCOVPlaybackController> playbackController = [sdkManager createFairPlayPlaybackControllerWithAuthorizationProxy:proxy]; playbackController.delegate = 자기; _playbackController = 재생 컨트롤러; [self.view addSubview:playbackController.view]; BCOVPlaybackService *playbackService = [[BCOVPlaybackService 할당] initWithAccountId:<계정 ID> policyKey:<정책 키>]; [playbackService findVideoWithVideoID:<video-id> 매개변수:@ {}완료:^(BCOVVideo *비디오, NSDictionary *jsonResponse, NSError *오류) { if(video) { [self.playbackController setVideos:@[video]]; } }];

레거시 비디오 클라우드 워크플로

레거시 Video Cloud 워크플로를 사용하는 경우 비디오를 재생하기 전에 FairPlay 인증서를 검색해야 합니다. 이 코드는 기본 설정 및 재생을 보여줍니다.

@property(비원자, 강력) id<BCOVPlaybackController> playbackController;
---
-
---
  // // FairPlay 자격 증명을 Brightcove에 등록하는 데 사용되는 // 애플리케이션 ID 및 게시자 ID를 사용하여 BCOVFPSBrightcoveAuthProxy 개체를 생성합니다. BCOVFPSBrightcoveAuthProxy *proxy = [[BCOVFPSBrightcoveAuthProxy 할당] initWithPublisherId:<pub-id> applicationId:<app-id> BCOVPlayerSDKManager *sdkManager = [BCOVPlayerSDKManager sharedManager]; // 인증 프록시를 사용하여 FairPlay 재생 컨트롤러를 만듭니다. id<BCOVPlaybackController> playbackController = [sdkManager createFairPlayPlaybackControllerWithAuthorizationProxy:proxy]; playbackController.delegate = 자기; _playbackController = 재생 컨트롤러; [self.view addSubview:playbackController.view]; // 프록시를 사용하여 FairPlay 애플리케이션 인증서를 검색합니다. // 프로덕션 앱에서는 앱이 시작되자마자 // 이 작업을 수행할 수 있습니다. [proxy retrieveApplicationCertificate:^(NSData * _Nullable applicationCertificate, NSError * _Nullable 오류) { if (applicationCertificate) { // Add the application certificate to the playback controller [playbackController addFairPlayApplicationCertificate:applicationCertificate identifier:kBCOVDefaultFairPlayApplicationCertificateIdentifier]; BCOVPlaybackService *playbackService = [[BCOVPlaybackService alloc] initWithAccountId:<account-id> policyKey:<policy-key>]; [playbackService findVideoWithVideoID:<video-id> parameters:@{}완료:^(BCOVVideo *비디오, NSDictionary *jsonResponse, NSError *오류) { if(video) { [playbackController setVideos:@[video]]; } }]; } }];

재생 컨트롤러에 여러 개의 FairPlay 애플리케이션 인증서를 추가할 수 있지만 레거시 재생의 경우 일반적으로 하나만 필요합니다. 동일한 재생 컨트롤러로 두 개의 다른 계정에서 비디오를 재생하는 경우 계정 ID를“식별자”매개변수.

이전 방법을 사용하여 재생 컨트롤러를 만드는 경우 해당 방법이 계속 작동합니다. 재생 컨트롤러가 생성되기 전이나 생성된 후에 FairPlay 애플리케이션 인증서를 검색할 수 있도록 하여 더 많은 유연성을 제공하기 위해 새로운 메서드가 생성되었으며 Dynamic Delivery의 경우’ t 응용 프로그램 인증서를 검색하거나 전달할 필요가 전혀 없습니다.

질문이 있거나 도움이 필요한 경우 Brightcove에 대한 지원 포럼이 있습니다. ’의 기본 플레이어 SDK [https://groups.google.com/forum/](https://groups.google.com/forum/)#!forum/brightcove-native-player-sdks .

신청 증명서

FairPlay로 보호된 비디오의 Playabck에는 콘텐츠와 연결된 애플리케이션 인증서가 필요합니다.

Video Cloud Dynamic Delivery를 사용하는 경우 필요에 따라 애플리케이션 인증서를 다운로드하여 사용합니다. 인증서는 재사용을 위해 캐시되므로 일반적으로 계정당 한 번만 다운로드됩니다.

레거시 Video Cloud 서비스를 사용하는 경우 초기화 방법과 함께 전달할 애플리케이션 인증서를 검색해야 합니다. NS BCOVFPSBrightcoveAuthProxy클래스는 fps.brightcove.com에서 이를 검색하는 편리한 방법을 제공합니다.

자신의 애플리케이션 인증서를 원격으로 저장하면 일반 NSURLSessionDataTask인증서 검색 NSData물체. 인증서를 받으면 다음을 사용하여 재생 컨트롤러에 인증서를 추가할 수 있습니다. [BCOVPlaybackController addFairPlayApplicationCertificate:식별자:] . 대부분의 경우 kBCOVDefaultFairPlayApplicationCertificateIdentifier식별자로.

타사 FairPlay 라이선스 서버 사용

NS BCOVFPSBrightcoveAuthProxy와 함께 작업하기 위한 것입니다 fps.brightcove.com . 다른 FairPlay 라이선스 서버를 사용해야 하는 경우 다음 중 하나를 구현할 수 있습니다. BCOVFPS인증 프록시라이센스 통신을 처리하는 프로토콜을 사용하거나 요구 사항이 간단한 경우 몇 가지를 무시할 수 있습니다. BCOVFPS인증 프록시속성:

/** * FairPlay 관련 라이선스 요청의 기본 URL입니다. 기본 URL은 *fps.brightcove.com을 가리킵니다. * nil로 설정하면 fps.brightcove.com을 가리키는 기본 NSURL이 다시 생성됩니다. */ @property(비원자, 강력, null_resettable) NSURL *fpsBaseURL; /** * FairPlay 관련 키 요청에 대한 키 요청 URL입니다. * 일반적으로 nil로 설정되며, 이 경우 키 요청 URL은 * Video Cloud Playback API 응답에서 검색됩니다. */ @property (비원자, 강력, null_resettable) NSURL *keyRequestURL;

NS fpsBaseURL속성은 응용 프로그램 인증서 요청에 대한 기본 URL을 설정하는 데도 사용됩니다. 인증서는 다음과 같이 요청됩니다.

[fpsBaseURL]/application_certificate/[publisherID]/[applicationID]

Video Cloud를 사용하든 자체 서버를 사용하든 BCOV비디오개체는 다음을 포함해야 합니다. BCOV 소스 URL이 일부 FairPlay 보호 콘텐츠를 가리키는 개체입니다. 보호되지 않은 비디오 URL은 여전히 FairPlay 워크플로에서 재생됩니다.
