iOS 및 tvOS용 Brightcove Player SDK용 사이드카 자막 플러그인, 버전 2.1.6.129

지원되는 플랫폼

iOS 8.0 이상.

tvOS 9.0 이상.

설치

Brightcove Player SDK용 사이드카 자막 플러그인은 iOS용 두 가지 설치 패키지를 제공합니다. 정적 라이브러리 프레임워크와 동적 프레임워크. tvOS의 경우 사이드카 자막 플러그인은 동적 프레임워크만 제공합니다.

코코아팟

당신이 사용할 수있는코코아팟 Brightcove Player SDK용 사이드카 자막 플러그인을 프로젝트에 추가합니다. 최신 정보를 찾을 수 있습니다 Brightcove-Player-SDK-사이드카자막포드 스펙여기 .

기본 포드 지정 Brightcove-Player-SDK-사이드카자막정적 라이브러리 프레임워크를 설치합니다. 동적 프레임워크를 설치하려면 다음을 사용하여 포드를 선언합니다. 동적하위 사양: Brightcove-Player-SDK-SidecarSubtitles/dynamic

정적 프레임워크 예:

pod 'Brightcove-Player-SDK-SidecarSubtitles'

동적 프레임워크 예:

pod 'Brightcove-Player-SDK-SidecarSubtitles/dynamic'

항상 최신 버전의 Brightcove 소프트웨어를 사용하려면 최신 마스터 podspec 리포지토리를 유지 관리해야 합니다. CocoaPods 1.0.0부터 podspec 리포지토리 업데이트는 더 이상 자동 기능이 아니므로 마스터 리포지토리를 업데이트하려면 명령줄에서 다음을 실행하세요.

포드 리포지토리 업데이트

설명서

Brightcove Player SDK용 사이드카 자막 플러그인을 프로젝트에 수동으로 추가하려면:

  1. 의 최신 버전을 설치하십시오. 브라이트코브 플레이어 SDK .
  2. BrightcoveSidecarSubtitles 플러그인의 최신 압축 릴리스를 다운로드하십시오. 릴리스 페이지 .
  3. 추가하다 BrightcoveSidecarSubtitles.framework당신의 프로젝트에.
  4. 에“빌드 설정”애플리케이션 대상의 탭에서“프레임워크 검색 경로”프레임워크에 대한 경로를 포함합니다. 프레임워크가 프로젝트와 다른 루트 디렉터리에 저장되지 않는 한 이 작업은 자동으로 수행되어야 합니다.
  5. (동적 프레임워크만 해당) “일반적인”응용 프로그램 대상의 탭에 추가‘ BrightcoveSidecarSubtitles.framework ’ ~로“임베디드 바이너리”부분.
  6. (동적 프레임워크만 해당) “빌드 단계”탭, 추가“스크립트 실행”명령으로 단계배시 $ {BUILT_PRODUCTS_DIR} /$ {FRAMEWORKS_FOLDER_PATH} /BrightcoveSidecarSubtitles.framework/strip-frameworks.sh . 확인하다“설치 시에만 스크립트 실행” . 이렇게 하면 앱 스토어 제출에 중요한 불필요한 아키텍처가 빌드에서 제거됩니다.
  7. (정적 프레임워크만 해당) “빌드 설정”응용 프로그램 대상의 탭에 추가 -오브제이C ~로“기타 링커 플래그”빌드 설정.

내장 PlayerUI 컨트롤 사용

위의 코드 스니펫은 컨트롤이 없는 비디오 플레이어를 나타냅니다. 다음과 같이 코드에 재생 컨트롤을 추가할 수 있습니다.

추적할 속성 추가 BCOVPUIPlayerView :

// PlayerUI's Player View @property (nonatomic) BCOVPUIPlayerView *playerView;

생성 BCOVPUIBasicControlView , 그리고 나서 BCOVPUIPlayerView . 여기에서 재생 컨트롤러(및 재생 컨트롤러가 재생하는 모든 비디오)를 컨트롤과 연결합니다. 레이아웃의 비디오 컨테이너와 일치하도록 플레이어 보기를 설정합니다( 당신의 비디오보기 ) 크기가 조정될 때.

BCOVPUIBasicControlView *controlView = [BCOVPUIBasicControlView basicControlViewWithVODLayout]; self.playerView = [[BCOVPUIPlayerView 할당] initWithPlaybackController:컨트롤러 옵션:nil controlsView:controlView]; // 부모 보기 크기 일치 self.playerView.frame = self.yourVideoView.bounds; self.playerView.autoresizingMask = UIViewAutoresizingFlexibleHeight | UIViewAutoresizingFlexibleWidth; // 비디오 보기에 BCOVPUIPlayerView를 추가합니다. [self.yourVideoView addSubview:self.playerView];

BCOVPUIPlayerView를 사용하는 경우 위의 한 줄도 제거해야 합니다.

[self.view addSubview:controller.view]; // PlayerUI를 사용할 때 더 이상 필요하지 않습니다.

다른 재생 컨트롤러와 함께 플레이어를 재사용하려는 경우 간단히 새 할당을 만들 수 있습니다.

self.playerView.playbackController = anotherPlaybackController;

자막 데이터 수동 채우기

BCOVSidecarSubtitle 플러그인은 자막 메타데이터 배열의 존재를 찾습니다. BCOV비디오개체 속성, 키 지정 kBCOVSSVideoPropertiesKeyTextTracks . 사용하는 경우 BCOV재생 서비스비디오를 검색하고 해당 비디오에 연결된 텍스트 트랙이 있는 경우 자동으로 채워집니다.

자신의 비디오를 제공하거나 Perform 고객인 경우 아래와 같이 데이터를 구조화해야 합니다.

NSArray *자막 = @[ @ { kBCOVSSTextTracksKeySource: ..., // required kBCOVSSTextTracksKeySourceLanguage: ..., // required kBCOVSSTextTracksKeyLabel: ..., // required kBCOVSSTextTracksKeyDuration: ..., // required/optional [1] kBCOVSSTextTracksKeyKind: kBCOVSSTextTracksKindSubtitles or kBCOVSSTextTracksKindCaptions, // required kBCOVSSTextTracksKeyDefault: ..., // optional kBCOVSSTextTracksKeyMIMEType: ..., // optional } , @ {...} , // 두 번째 텍스트 트랙 사전 @ {...} , // 세 번째 텍스트 트랙 사전 ]; BCOVVideo *비디오 = [BCOVVideo 할당] initWithSource:<source> cuePoints:<cuepoints> 속성:@ { kBCOVSSVideoPropertiesKeyTextTracks:subtitles } ];

NS kBCOVSSTextTracksKeySource키는 자막 트랙의 소스 URL을 보유하며 WebVTT URL 또는 M3U8 재생 목록 URL로 제공될 수 있습니다.

WebVTT 파일에는“ .vtt ”확장자가 있어야 하며 M3U8 파일에는“ .M3U8 ”확대. 이러한 규칙을 따를 수 없는 경우 다음을 포함해야 합니다. kBCOVSSTextTracksKeySourceType키를 입력하고 다음 중 하나를 지정합니다. kBCOVSSTextTracksKeySourceTypeWebVTTURL또는 kBCOVSSTextTracksKeySourceTypeM3U8URL URL에서 참조하는 파일 유형을 나타냅니다.

알려진 문제

  • 360도 동영상을 볼 때 자막이 표시되지 않습니다.

  • 이 플러그인은 현재 iOS용 Brightcove Brightcove Player SDK용 Widevine 플러그인과의 통합을 지원하지 않습니다.

  • Brightcove Playback API에서 비디오를 검색할 때 변환에 마스터 M3U8 재생 목록이 포함되어야 합니다. Sidecar 자막 플러그인은 단일 변환 M3U8 재생 목록에서 작동하지 않습니다.

  • 플러그인에 자막 재생 목록을 제공하고 해당 자막 재생 목록에 404로 응답하는 웹 vtt 파일에 대한 링크가 포함되어 있으면 재생이 실패합니다. 이것은 애플의 버그입니다. ’ s AVPlayer.
