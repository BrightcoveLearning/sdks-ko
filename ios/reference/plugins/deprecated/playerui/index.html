BCOVPlayerUI 참조

iOS용 Brightcove Player SDK용 BCOVPlayerUI 플러그인, 버전 1.0.4.143

지원되는 플랫폼

iOS 7.0 이상.

설치

Brightcove PlayerUI 플러그인은 iOS용 두 가지 설치 패키지인 정적 라이브러리 프레임워크와 동적 프레임워크를 제공합니다. 정적 라이브러리 대상은 iOS 7에서의 배포를 지원하는 반면 동적 프레임워크는 iOS 8 이상만 지원합니다.

코코아팟

당신이 사용할 수있는코코아팟 Brightcove Player SDK용 PlayerUI 플러그인을 프로젝트에 추가합니다. 최신 정보를 찾을 수 있습니다 Brightcove-Player-SDK-Player-UI포드 스펙여기 .

정적 프레임워크 예:

포드 'Brightcove-Player-SDK-Player-UI'

동적 프레임워크 예:

pod 'Brightcove-Player-SDK-Player-UI/dynamic'

설명서

Brightcove Player SDK용 PlayerUI 플러그인을 프로젝트에 수동으로 추가하려면:

  1. 의 최신 버전을 설치하십시오. 브라이트코브 플레이어 SDK .
  2. 플러그인의 최신 압축 릴리스를 다운로드하십시오. 릴리스 페이지 .
  3. 추가‘ BrightcovePlayerUI.framework ’당신의 프로젝트에.
  4. 에“빌드 설정”애플리케이션 대상의 탭에서“프레임워크 검색 경로”프레임워크에 대한 경로를 포함합니다. 프레임워크가 프로젝트와 다른 루트 디렉터리에 저장되지 않는 한 이 작업은 자동으로 수행되어야 합니다.
  5. 에“빌드 단계”응용 프로그램 대상의 탭에서 다음을 추가하십시오. “라이브러리와 바이너리 연결”단계:

  6. (동적 프레임워크만 해당) “일반적인”응용 프로그램 대상의 탭에 추가‘ BrightcovePlayerUI.framework ’ ~로“임베디드 바이너리”부분.

  8. (정적 프레임워크만 해당) “빌드 설정”응용 프로그램 대상의 탭에 추가 -오브제이C ~로“기타 링커 플래그”빌드 설정.
  9. (Static Framework만 해당) 파일 추가 bcovpuiiconfont.ttf ~로부터 BrightcovePlayerUI.framework글꼴 파일이 앱 번들에 복사되도록 프로젝트 목록에 직접 번들로 묶습니다. 결국 글꼴 파일은 앱과 같은 수준으로 끝나야 합니다. ’ NS 정보.plist파일. 글꼴 파일은 일부 BrightcovePlayerUI 인터페이스 요소를 제공하지만 plist 자체에 나열할 필요는 없습니다.

수입품

iOS용 Brightcove Player SDK는 몇 가지 다른 방법으로 코드로 가져올 수 있습니다. @import BrightcovePlayerUI; , #import <BrightcovePlayerUI/BrightcovePlayerUI.h>또는 #import <BrightcovePlayerUI/[특정 클래스].h> .

빠른 시작

사용법은 비교적 간단합니다. 이 예에서는 일반 주문형 비디오 스트림에 적합한 기본 레이아웃을 만듭니다.

허락하다’ s 이 코드를 단계로 분해하여 좀 더 쉽게 소화할 수 있습니다.

  1. 기본 설정으로 재생 컨트롤러를 만듭니다.
  2. 재생 컨트롤러를 사용하여 플레이어 보기를 만듭니다. 기본 설정으로 플레이어 보기를 인스턴스화합니다. 비디오 컨테이너 보기의 크기와 일치하도록 플레이어 보기를 설정합니다.
  3. 비디오를 검색하고 재생합니다.

플러그인 동작 사용자 정의

BCOVPlayerUI 플러그인은 즉시 사용 가능한 간편한 경험을 제공합니다. 그러나, 그것은’기본 동작을 변경할 수 있습니다.

BCOVPUIPlayerViewOptions

NS BCOVPUIPlayerViewOptions클래스를 사용하면 초기화 시 일부 BCOVPlayerUI 동작을 사용자 지정할 수 있습니다. 다음을 사용자 지정할 수 있습니다.

  • 점프백 간격뒤로 점프 버튼을 누르면 플레이어가 뒤로 점프하는 시간(초)입니다.
  • hideControls간격마지막 터치 이벤트 후 컨트롤이 숨겨지기 전의 시간(초)입니다.
  • hideControlsAnimationDuration컨트롤이 숨김 상태로 전환되는 데 걸리는 시간(초)입니다.
  • showControlsAnimationDuration컨트롤이 표시될 때까지 걸리는 시간(초)입니다.
  • 프레젠테이션 보기 컨트롤러모달 및 전체 화면과 같은 것을 표시하는 데 사용할 뷰 컨트롤러입니다.

옵션은 다음 방법을 사용하여 설정할 수 있습니다.

 BCOVPlayerSDKManager *manager = [BCOVPlayerSDKManager sharedManager]; id<BCOVPlaybackController> controller = [manager createPlaybackController]; BCOVPUIPlayerViewOptions *options = [[BCOVPUIPlayerViewOptions alloc] init]; options.jumpBackInterval = 5; BCOVPUIPlayerView *playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController:playbackController options:options];

제공되는 레이아웃

다양한 유형의 비디오를 지원하기 위해 세 가지 레이아웃이 제공됩니다.

  • BCOVPUIControlLayout 기본VODControlLayout일반 주문형 비디오 스트림의 기본 레이아웃입니다.

  • BCOVPUIControlLayout 기본LiveControlLayout라이브 비디오를 대상으로 하는 레이아웃입니다.

  • BCOVPUIControlLayout 기본LiveDVRControlLayout DVR 컨트롤이 있는 라이브 비디오 스트림을 대상으로 합니다.

일반적으로 새 레이아웃을 즉시 설정합니다. BCOVPUIPlayerView생성되지만 언제든지 새 레이아웃을 설정할 수도 있습니다. 예를 들어 다음과 같이 새 VOD 레이아웃을 설정할 수 있습니다.

playerview.controlView.layout = [BCOVPUIControlLayout 기본VODControlLayout]

사용자 정의 레이아웃

기본 레이아웃 외에도 새로운 BCOVPUIControlLayout나만의 디자인으로.

먼저 다음을 사용하여 레이아웃에 들어갈 컨트롤을 만듭니다. BCOVPUIBasicControlView 레이아웃ViewWithControlFromTag:width:elasticity: . 각 컨트롤은 BCOVPUILayoutView제어 간격을 결정합니다. 당신은 설정할 수 있습니다너비각 레이아웃 보기의 기본 너비(컨트롤 유형 기반)로 변경하거나 사용자 고유의 너비를 지정할 수 있습니다.

NS 탄력인수는 컨트롤을 포함하는 레이아웃 보기가 컨트롤 막대를 채우기 위해 너비를 조정하는 정도를 결정합니다. 탄력성이 0이면 레이아웃 보기의 크기가 고정됩니다. 탄력성 값이 0보다 크면 해당 컨트롤 막대의 다른 모든 탄력적 보기에 비해 사용 가능한 공간을 채우기 위해 레이아웃 보기가 얼마나 커질지 결정합니다. 탄력성 값이 2.0인 레이아웃 보기는 탄력성 값이 1.0인 레이아웃 보기보다 두 배 빠르게 성장합니다. 일반적으로 레이아웃 보기 행에는 탄력성이 0보다 큰 컨트롤이 하나 이상 있습니다.

다음은 다양한 기본 컨트롤을 만드는 예입니다.

BCOVPUILayoutView *playbackLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagButton재생 너비:kBCOVPUILayoutUseDefaultValue 탄력성:0.0]; BCOVPUILayoutView *jumpBackButtonLayoutView = [BCOVPUIBasicControlView 레이아웃ViewWithControlFromTag:BCOVPUIViewTagButtonJumpBack 너비:kBCOVPUILayoutUseDefaultValue 탄력성:0.0]; BCOVPUILayoutView *closedCaptionLayoutView = [BCOVPUIBasicControlView 레이아웃ViewWithControlFromTag:BCOVPUIViewTagButtonClosedCaption 너비:kBCOVPUILayoutUseDefaultValue 탄력성:0.0]; BCOVPUILayoutView *screenModeLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagButtonScreenMode 너비:kBCOVPUILayoutUseDefaultValue 탄력성:0.0]; BCOVPUILayoutView *externalRouteLayoutView = [BCOVPUIBasicControlView 레이아웃ViewWithControlFromTag:BCOVPUIViewTagViewExternalRoute 너비:kBCOVPUILayoutUseDefaultValue 탄력성:0.0]; BCOVPUILayoutView *spacerLayoutView1 = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagViewEmpty 너비:1.0 탄성:1.0]; BCOVPUILayoutView *spacerLayoutView2 = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagViewEmpty 너비:1.0 탄성:1.0]; BCOVPUILayoutView *logoLayoutView1 = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagViewEmpty 너비:80.0 탄성:1.0]; externalRouteLayoutView.removed = 예;

자신의 보기(로고, 컨트롤, 아무것도 없음 등)를 배치할 수 있는 빈 레이아웃 보기를 만들 수도 있습니다. 이 코드는 위에서 만든 logoLayoutView1 안에 UIImage 로고를 배치하는 방법을 보여줍니다.

// 컨트롤 막대에 표시할 이미지 보기 내부에 로고 이미지를 만듭니다. UIImage *logoImage1 = [UIImage imageNamed:@"myLogo"]; UIImageView *logoImageView1 = [[UIImageView 할당] initWithImage:logoImage1]; logoImageView1.autoresizingMask = UIViewAutoresizingFlexibleWidth | UIViewAutoresizingFlexibleHeight; logoImageView1.contentMode = UIViewContentModeScaleAspectFit; logoImageView1.frame = logoLayoutView1.frame; // 빈 레이아웃 뷰에 이미지 뷰를 추가합니다. [logoLayoutView1 addSubview:logoImageView1];

이제 레이아웃 보기에 패키지된 컨트롤 모음이 있으므로 이를 배열로 정렬할 수 있습니다. 각 컨트롤은 한 줄의 컨트롤, 즉 컨트롤 막대를 나타냅니다. 세로 및 가로 방향에 대해 다른 레이아웃을 가질 수 있으므로 일반적으로 두 개의 서로 다른 컨트롤 막대 배열을 설정합니다.

가로 방향에 대한 이 표준 레이아웃에서는 컨트롤을 한 줄에 정렬한 다음 해당 배열을 전체 구성을 나타내는 다른 배열에 넣습니다.

 NSArray *standardLayoutLine1 = @[ playbackLayoutView, jumpBackButtonLayoutView, currentTimeLayoutView, timeSeparatorLayoutView, durationLayoutView, progressLayoutView, spacerLayoutView1, logoLayoutView1, spacerLayoutView2, closedCaptionLayoutView, screenModeLayoutView, externalRouteLayoutView ]; NSArray *standardLayoutLines = @[ standardLayoutLine1 ];

컴팩트 레이아웃에서 세로 방향의 경우 두 개의 컨트롤 배열을 만든 다음 이 두 배열을 컴팩트 레이아웃을 나타내는 다른 배열로 패키징합니다.

각 레이아웃의 대부분의 컨트롤에 정확히 동일한 개체를 사용했습니다. 이 작업이 완료되고 세로 방향과 가로 방향 사이를 전환하면 개체가 부드러운 애니메이션을 사용하여 새 위치로 이동합니다.

 NSArray *compactLayoutLine1 = @[ currentTimeLayoutView, progressLayoutView, durationLayoutView ]; NSArray *compactLayoutLine2 = @[ playbackLayoutView, jumpBackButtonLayoutView, spacerLayoutView1, closedCaptionLayoutView, screenModeLayoutView, externalRouteLayoutView, logoLayoutView2 ]; NSArray *compactLayoutLines = @[ compactLayoutLine1, compactLayoutLine2 ];

마지막으로 전체 너비와 컴팩트 너비의 두 가지 레이아웃 구성이 있으므로 새 레이아웃을 만들 수 있습니다. BCOVPUIControlLayout개체 및 플레이어에서 설정’ s 컨트롤 뷰.

BCOVPUIControlLayout *customLayout = [[BCOVPUIControlLayout 할당] initWithStandardControls:standardLayoutLines compactControls:compactLayoutLines]; playerView.controlView.layout = 사용자 정의 레이아웃;

자주 표시하거나 숨겨야 하는 컨트롤이 있는 경우 다음을 설정할 수 있습니다. 제거됨해당 컨트롤의 속성’ s 레이아웃 보기. 컨트롤을 변경한 경우 다음으로 전화하십시오. setNeedsLayout플레이어뷰에서’ s controlView:

 logoLayoutView1.removed = YES; [playerView.controlsView setNeedsLayout];

또한 몇 가지 일반 BCOVPUIControlLayout속성:

  • 컨트롤바 높이각 컨트롤 행의 크기를 설정합니다.
  • 수평 항목 간격각 사이의 간격을 설정합니다. BCOVPUILayoutView각 컨트롤 바에서
  • 컴팩트 레이아웃최대폭사용되는 컨트롤 집합을 결정합니다. 컨트롤 뷰가 다음보다 작은 경우컴팩트 레이아웃최대폭 , 컴팩트 컨트롤 세트가 사용되며, 그렇지 않으면 표준 컨트롤이 사용됩니다.

표시되는 컨트롤 집합을 변경하려면 새 컨트롤을 만들고 설치해야 합니다. BCOVPUIControlLayout .

자주 묻는 질문

모든 Brightcove Player SDK 플러그인에서 작동합니까?

이 릴리스는 광고 없이 기본 VOD, 라이브 및 라이브 DVR 재생을 지원합니다. 향후 릴리스에서 다른 플러그인에 대한 지원을 추가할 예정입니다.

위에서 언급한 것 이외의 속성을 어떻게 사용자 정의할 수 있습니까?

우리의 목표는’사용자 정의하기 쉽고 많은 기능과 동작을 기본적으로 제공합니다. 다음 릴리스에서는 새로운 API와 더 많은 문서로 더 많은 사용자 정의를 다룰 것입니다. 찾고 있는 기능이 있으면 포럼에서 알려주십시오.

향후 버전에서 주요 API 변경 사항이 있습니까?

정말 필요한 경우가 아니면 변경하지 않도록 최선을 다할 것입니다. 그러나 API를 변경해야 하는 경우 CHANGELOG에서 명확한 마이그레이션 단계를 제공합니다.
