iOS 참조용 Brightcove 기본 플레이어

iOS용 Brightcove Player SDK용 Omniture(Adobe Marketing Cloud - Adobe Analytics) 플러그인, 버전 6.9.0.1697

요구 사항

  • 엑스코드 11.0+
  • 호

지원되는 플랫폼

iOS 11.0 이상.

설치

Omniture 플러그인은 Adobe Marketing Cloud v4.14.1 및 Adobe Video Heartbeat v2.0.1을 지원하는 정적 라이브러리 프레임워크입니다. Marketing Cloud 및 Video Heartbeat 라이브러리아니다이 SDK에 포함되어 있으며 프로젝트에 수동으로 추가해야 합니다. 라이브러리 다운로드 지침은 아래에 나와 있습니다.

코코아팟

당신이 사용할 수있는코코아팟 Omniture 플러그인을 프로젝트에 추가하려면 버전 1.0 이상을 사용하십시오. 최신 정보를 찾을 수 있습니다 Brightcove-Player-Omniture포드 스펙여기 . 이 사양을 사용하려면 Podfile 상단에 다음을 추가하세요. 원천 ' https://github.com/brightcove/BCOVSpecs.git' .

정적 프레임워크 예:

```배시 소스‘ https://github.com/brightcove/BrightcoveSpecs.git'

표적‘ MyOmniturePlayer ’포드‘ Brightcove-Player-Omniture ’끝 ```

설치를 업데이트할 때’ CococaPods 마스터 리포지토리를 업데이트하는 것처럼 BrightcoveSpecs 리포지토리의 로컬 복사본을 새로 고치는 것이 좋습니다.

일반적으로 사용하는 경우포드 업데이트터미널에서는 자동으로 발생하거나 다음을 사용하여 명시적으로 업데이트할 수 있습니다. 포드 리포지토리 업데이트 .

설명서

Brightcove Player SDK용 Omniture 플러그인을 프로젝트에 수동으로 추가하려면:

  1. 의 최신 버전을 설치하십시오. 브라이트코브 플레이어 SDK .
  2. 플러그인의 최신 압축 릴리스를 다운로드하십시오. 릴리스 페이지 .
  3. 추가 BrightcoveAMC.framework프로젝트. Frameworks 폴더를 마우스 오른쪽 버튼으로 클릭하고 선택하여 이 작업을 수행할 수 있습니다. “파일 추가”옵션을 선택하고 BrightcoveAMC.framework저장된 경로에서.
  4. 에“빌드 설정”애플리케이션 대상의 탭에서“프레임워크 검색 경로”프레임워크에 대한 경로를 포함합니다. 프레임워크가 프로젝트와 다른 루트 디렉터리에 저장되지 않는 한 이 작업은 자동으로 수행되어야 합니다.
  5. 에“빌드 단계”응용 프로그램 대상의 탭에서 다음을 추가하십시오. “라이브러리와 바이너리 연결”단계:
    • BrightcoveAMC.framework
  6. 에“빌드 설정”애플리케이션 대상의 탭:
    • 다음을 확인하십시오. -오브제이C에 추가되었습니다“기타 링커 플래그”빌드 설정.
  7. Adobe 다음 Adobe Marketing Cloud 다운로드’ NS 지침그리고 추가 AdobeMobileLibrary.a당신의 프로젝트에.
  8. Adobe 다음 Adobe Video Heartbeat 다운로드’ NS 지침그리고 추가 VideoHeartbeat.a당신의 프로젝트에.
  9. Marketing Cloud 및 Video Heartbeat 헤더 폴더를 프로젝트의 헤더 검색 경로 설정에 추가하십시오.

빠른 시작

BrightcoveAMC는 iOS용 브라이트코브 플레이어 SDK및 Adobe 비디오 추적 포함어도비 비디오 하트비트 v2그리고어도비 미디어 추적 v4 . BrightcoveAMC는 비디오 준비, 비디오 완료, 비디오 재생 및 비디오 일시 중지와 같은 기본 비디오 동작을 추적합니다.

클라이언트 구현

Adobe Video Heartbeat를 설정하려면 Omniture 플러그인 클라이언트가 BCOVAMCVideoHeartbeatConfigurationPolicy의 인스턴스를 구현해야 합니다. Adobe Media Tracking을 설정하려면 BCOVAMCMediaSettingPolicy의 인스턴스가 필요합니다. 이것들정책각 재생 세션에 대해 ADBMediaHeartbeatConfig 또는 ADBMediaSettings 개체를 사용자 지정할 수 있습니다.

비디오 하트비트 v2

이 예에서는 비디오 하트비트 추적을 사용합니다.

[1] // VHB 구성 정책 객체를 생성합니다. BCOVAMCVideoHeartbeatConfigurationPolicy videoHeartbeatConfigurationPolicy = ^ADBMediaHeartbeatConfig *(id<BCOVPlaybackSession> 세션) { ADBMediaHeartbeatConfig *configData = [[ADBMediaHeartbeatConfig alloc] init]; configData.trackingServer = <adobe-assigned_tracking_server>"; configData.channel = <cutomize_sample_channel>; configData.appVersion = <app_version>; configData.ovp = <online_video_platform>; configData.playerName = <player_name>; configData.ssl = <YES | NO>; // Set debugLogging to true to activate debug tracing. Remove it in production. configData.debugLogging = YES; return configData; } ; [2] // w1w VHB 구성 정책 개체에서 Brightcove AMC 분석 정책 개체를 만듭니다. BCOVAMCAnalyticsPolicy *heartbeatPolicy = [[BCOVAMCAnalyticsPolicy alloc] initWithHeartbeatConfigurationPolicy:videoHeartbeatConfigPolicy]; BCOVAMCSessionConsumer *sessionConsumer = [BCOVAMCSessionConsumer heartbeatAnalyticsConsumerWithPolicy:heartbeatPolicy 대리자:self]; BCOVPlayerSDKManager *관리자 = [BCOVPlayerSDKManager 공유 관리자]; id<BCOVPlaybackController> 컨트롤러 = [매니저 createPlaybackController]; controller.delegate = 자기; [self.view addSubview:controller.view]; [3] // Brightcove AMC 세션 소비자를 재생 컨트롤러에 추가합니다. [컨트롤러 addSessionConsumer:sessionConsumer]; [4] // 동영상을 찾아서 재생합니다. NSString *policyKey = <귀하의 정책 키>; NSString *accountId = <귀하의 계정 ID>; NSString *videoID = <당신의 비디오 ID>; BCOVPlaybackService *service = [[BCOVPlaybackService 할당] initWithAccountId:accountId policyKey:policyKey]; [서비스 findVideoWithVideoID:videoID 매개변수:nil 완료:^(BCOVVideo *video, NSDictionary *jsonResponse, NSError *error) { [controller setVideos:@[ video ]]; [controller play]; } ];
  1. 각 재생 세션이 시작될 때 호출될 비디오 하트비트 구성 정책 블록을 만듭니다. 정책을 통해 현재 세션을 기반으로 구성 데이터를 사용자 지정할 수 있습니다.
  2. 구성 정책 블록을 사용하여 BCOVAMCAnalyticsPolicy 인스턴스를 생성하고 초기화합니다. -[initWithHeartbeat 구성 정책:] . BCOVAMCAnalyticsPolicy 개체는 BCOVAMCSessionConsumer를 만드는 데 사용됩니다.
  3. 재생 컨트롤러가 생성된 후 세션 소비자 추가 메서드를 호출합니다. -[추가세션 소비자:] , AMC 세션 소비자를 추가합니다.

마케팅 클라우드

이 예에서는 미디어 추적을 사용합니다.

[1] BCOVAMCMediaSettingPolicy mediaSettingPolicy = ^ADBMediaSettings *(id<BCOVPlaybackSession> 세션) { ADBMediaSettings *settings = [ADBMobile mediaCreateSettingsWithName:<cutomize_setting_name> [2] length:0 playerName:<cutomize_player_name> playerID:<cutomize_player_ID>]; [3] settings.milestones = @"25,50,75"; // a customization. return settings; } ; [4] BBCOVAMCAnalyticsPolicy *mediaPolicy = [[BCOVAMCAnalyticsPolicy alloc] initWithMediaSettingsPolicy:mediaSettingPolicy]; BCOVAMCSessionConsumer *sessionConsumer = [BCOVAMCSessionConsumer mediaAnalyticsConsumerWithPolicy:mediaPolicy 대리자:자신]; BCOVPlayerSDKManager *관리자 = [BCOVPlayerSDKManager 공유 관리자]; id<BCOVPlaybackController> 컨트롤러 = [매니저 createPlaybackController]; controller.delegate = 자기; [self.view addSubview:controller.view]; [5] [컨트롤러 addSessionConsumer:sessionConsumer]; NSString *policyKey = <귀하의 정책 키>; NSString *accountId = <귀하의 계정 ID>; NSString *videoID = <당신의 비디오 ID>; BCOVPlaybackService *service = [[BCOVPlaybackService 할당] initWithAccountId:accountId policyKey:policyKey]; [서비스 findVideoWithVideoID:videoID 매개변수:nil 완료:^(BCOVVideo *video, NSDictionary *jsonResponse, NSError *error) { [controller setVideos:@[ video ]]; [controller play]; } ];
  1. 각 재생 세션이 시작될 때 호출될 미디어 설정 정책 블록을 만듭니다. 이 정책을 사용하면 현재 세션을 기반으로 미디어 설정을 사용자 지정할 수 있습니다.
  2. ADBMobile의 클래스 메소드로 ADBMediaSettings 인스턴스를 생성할 때 +[mediaCreateSettingsWithName:length:playerName:playerID:] , 설정할 수 있습니다비디오 길이에게 0 . Omniture 플러그인은 나중에 업데이트합니다.
  3. 이정표와 같은 선택적 설정을 추가합니다.
  4. 미디어 설정 정책 블록을 사용하여 BCOVAMCAnalyticsPolicy 인스턴스를 생성하고 초기화합니다. -[initWithMediaSettingsPolicy:] . 정책 개체는 다음과 같이 Adobe 미디어 추적을 위한 AMC 세션 소비자를 만드는 데 사용됩니다. +[mediaAnalyticsConsumerWithPolicy:대리인:] .
  5. 재생 컨트롤러가 생성된 후 세션 소비자 추가 메서드를 호출합니다. -[추가세션 소비자:] , AMC 세션 소비자를 추가합니다.
