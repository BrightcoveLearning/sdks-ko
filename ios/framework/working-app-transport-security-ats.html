개요

iOS 9 출시와 함께 Apple은 새로운 기능을 도입했습니다. 앱 전송 보안(ATS)앱과 웹 서비스 간의 보안 연결을 적용합니다. 가까운 장래에 앱 검토 프로세스의 일부로 ATS 예외에 대한 합당한 근거를 제공하지 않는 한 앱 스토어의 모든 앱은 ATS를 활성화해야 합니다.

곧 모든 App Store 앱에서 App Transport Security를 사용해야 합니다. 더 이상 App Store 앱용 ATS를 비활성화할 수 없습니다.
  • 자세한 내용은 Apple은 2016년 말까지 iOS 앱에 HTTPS 연결을 요구할 것입니다.문서.
  • 따르다 Apple의 뉴스 및 업데이트 ATS가 언제 필요한지 알아보십시오.

ATS 제한 사항은 iOS 9.0 이상을 대상으로 하는 애플리케이션에 적용됩니다. 앱 스토어의 기존 앱은 영향을 받지 않지만 새 앱을 게시하거나 기존 앱을 업데이트할 때 ATS 제한을 적용해야 합니다.

Brightcove 에코시스템은 ATS 제한을 지원합니다. 대부분의 경우 ATS가 활성화된 앱을 빌드해야 합니다. 비보안 URL에 연결해야 하는 경우 Apple의 App Store 검토에 근거와 함께 ATS 예외를 제출할 수 있습니다.

다음 중 하나를 선택합니다.

  • ATS가 활성화된 앱 빌드
  • ATS 예외 사용

최후의 수단으로 ATS 요구 사항을 비활성화할 수 있습니다. 이 옵션은 곧 중단될 예정입니다.

  • ATS 요구 사항 비활성화

ATS가 활성화된 앱 빌드

앱과 웹 서비스 간에 HTTPS를 통한 보안 연결을 사용하는 경우 앱에 대해 App Transport Security를 활성화할 수 있습니다.

ATS에서 정의한 보안 연결에는 TLS v1.2, 순방향 보안 및 SHA-2 인증서가 포함됩니다. 전체 사양은 다음을 참조하십시오. ATS를 사용하여 연결하기 위한 요구 사항문서.

ATS가 활성화된 앱을 빌드하려면 다음을 수행하십시오.

  1. 보안 통신을 위해 CDN을 활성화하십시오.

    • Video Cloud 고객(하우스 CDN 또는 명명된 CDN 사용 여부)은 해당 Brightcove 계정 관리자에게 문의하여 계정이 HTTPS를 통해 제공되도록 구성되었는지 확인해야 합니다.

    • 원격 자산을 사용하는 경우 CDN을 적절하게 구성해야 할 수 있습니다.

  2. 보안 통신을 사용하여 미디어 콘텐츠를 전달합니다. 여기에는 비디오, 광고, 캡션, 포스터 이미지, 썸네일 및 기타 타사 API 연결이 포함됩니다.

    • 비디오 클라우드 고객: Playback API를 사용하여 각 자산에 대한 보안 프로토콜을 검색합니다.

      기본적으로 iOS용 Brightcove 기본 SDK는 소스 선택 정책을 사용하여 HTTP 소스보다 HTTPS를 선택하므로 ATS가 활성화된 앱을 빌드할 수 있습니다.

    • 브라이트코브 플레이어 고객: 미디어 콘텐츠가 HTTPS를 통해 전달되는지 확인합니다.

ATS 예외 사용

앱과 웹 서비스 간에 비보안 연결을 사용해야 하는 경우 ATS 예외를 사용해 볼 수 있습니다. ATS 요구 사항의 이유는 사용자가 앱을 사용할 때 높은 수준의 보안을 유지할 수 있도록 하기 위함입니다. 따라서 ATS 예외를 조사하기 전에 앱이 보안 연결을 사용할 수 없는지 확인하십시오.

ATS 요구 사항을 우회하는 데 사용할 수 있는 특정 키가 있습니다. 이러한 예외는 추가 App Store 검토를 유발하며 합리적인 근거를 제공해야 합니다. 자세한 내용은 ATS에 대한 앱 스토어 리뷰문서.

이미 암호화된 콘텐츠(예: FairPlay)에 대한 예외에 대한 정당성을 얻을 수 있지만 가능하면 HTTPS를 사용하는 것이 가장 좋습니다.

ATS 요구 사항 비활성화

iOS 9 SDK 이상을 대상으로 하는 애플리케이션을 빌드할 때 ATS를 비활성화하도록 선택할 수 있습니다. 앱에서 ATS 요구 사항을 비활성화하려면 다음 단계를 따르세요.

곧 모든 App Store 앱에서 App Transport Security를 사용해야 합니다. 더 이상 App Store 앱용 ATS를 비활성화할 수 없습니다.
  • 자세한 내용은 Apple은 2016년 말까지 iOS 앱에 HTTPS 연결을 요구할 것입니다.문서.
  • 따르다 Apple의 뉴스 및 업데이트 ATS가 언제 필요한지 알아보십시오.

  1. Xcode에서 오른쪽 클릭정보.plist파일을 열고 다음과 같이 엽니다. 소스 코드 .

  2. 다음 코드를 복사합니다.

    <key>NSAppTransportSecurity</key>
<dict>
    <key>NSAllowsArbitraryLoads</key>
    <true/>
</dict>

  3. 에서정보.plist파일, 닫기 전에</dict>태그에 위의 코드를 붙여넣습니다.

  4. Xcode에서 열 때정보.plist파일로속성 목록 , 이제 plist 사전에 이 항목이 표시되어야 합니다.

ATS를 비활성화해도 기존 HTTPS 연결은 변경되지 않습니다. 그것은 단순히 iOS가 http 통신에 추가 보안 요구 사항을 적용하는 것을 방지합니다.
