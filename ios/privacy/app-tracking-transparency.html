소개

iOS 14.5에는 앱 추적 투명성이라는 새로운 개인 정보 보호 기능이 있습니다. 앱에서 이 기능을 사용하여 사용자에게 추적 권한을 요청할 수 있습니다. 사용자가 추적을 거부하면 앱은 광고 타겟팅을 위한 활동에 대한 데이터를 공유하기 위해 Apple의 IDFA(광고주용 ID) 또는 해시된 이메일 주소와 같은 기타 식별자를 사용할 수 없습니다.

앱이 최종 사용자 데이터를 수집하고 앱 및 웹 사이트 전반에서 추적 목적으로 다른 회사와 공유하는 경우 다음을 사용해야 합니다. AppTrackingTransparency뼈대.

자세한 내용은 다음을 참조하세요.

  • 앱 스토어: 사용자 개인 정보 및 데이터 사용
  • 앱 추적 투명성
  • IMA SDK: SKAdNetwork에서 전환 추적 가능
  • INVIDI 펄스 문서
iOS 및 tvOS용 기본 SDK는 다음을 통해 앱 추적 투명성을 지원합니다. Brightcove IMA 및 Pulse 플러그인 .

요구 사항

앱 추적 투명성을 위해서는 다음 요구 사항이 필요합니다.

플랫폼

시스템 요구 사항은 다음과 같습니다.

  • iOS 또는 tvOS 버전 14.5 이상

브라이트코브 네이티브 SDK

최신 버전의 Native SDK를 사용하는 것이 좋습니다.

  • iOS 또는 tvOS 버전 6.8.0 이상용 기본 SDK
  • 자세한 내용은 SDK 출시 정보

구현

아래에 언급된 샘플 앱은 앱 추적 투명성 경고를 표시하여 앱에서 IDFA(광고주 ID)를 사용할 수 있는지 여부를 결정하는 방법을 보여줍니다.

  1. 다음 샘플 앱 중 하나를 열고 검토합니다.

    • 기본IMAP 플레이어
    • 기본펄스플레이어
  2. 열기BaseViewController , viewDidLoad방법.

  3. iOS 14의 경우 SDK는 다음 값을 확인합니다. ATTrackingManager.trackingAuthorizationStatus . 값이ATTrackingManagerAuthorizationStatusAuthorized , SDK는 IDFA를 사용합니다.

    if (@available(iOS 14, *))
  {
  __weak typeof(self) weakSelf = self;
  [ATTrackingManager requestTrackingAuthorizationWithCompletionHandler:^(ATTrackingManagerAuthorizationStatus status) {
      __strong typeof(weakSelf) strongSelf = weakSelf;
      dispatch_async(dispatch_get_main_queue(), ^{
          // Tracking authorization completed. Start loading ads here.
          [strongSelf videoLibrary];
          
          [strongSelf setupPlayerView];
          [strongSelf setupPlaybackController];
          [strongSelf requestVideo];
      });
  }];
  4. 이전 iOS 버전의 경우 SDK는 다음 값을 사용합니다. ASIdentifierManager.sharedManager.isAdvertisingTrackingEnabled IDFA를 사용해야 하는지 여부를 결정합니다.
