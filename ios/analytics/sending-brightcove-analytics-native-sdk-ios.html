분석 통합

Brightcove 플레이어를 사용하는 경우catalog비디오를 검색하고 재생하기 위해 SDK에 포함된 분석 데이터는 자동으로 수집되어 Video Cloud 분석 모듈에 나타납니다.

재정의하면BCOVVideo클래스를 사용하거나 Brightcove 플레이어를 사용하지 않고catalog , 계정 ID를 Video Cloud Analytics로 보내야 합니다.

속성 사용자 정의

iOS용 Brightcove Mobile SDK 버전 4.1.8부터 계정 ID, 대상 URI 및 소스 URI에 대한 사용자 지정 값을 앱에서 Brightcove Analytics로 보낼 수 있습니다.

계정 ID가 필요하지만 Brightcove 플레이어를 사용할 때 자동으로 설정되고catalog .

원하는 경우 대상 및 소스 URI를 설정할 수 있습니다. 기본값은 응용 프로그램 패키지 이름을 기반으로 합니다.

이러한 속성은 귀하의ViewController구현 파일. 이것은 일반적으로 생성을 위한 호출 후에 수행됩니다. PlaybackController :

id<BCOVPlaybackController> controller =
    [[BCOVPlayerSDKManager sharedManager] createPlaybackController];

controller.analytics.account = your accountId;
controller.analytics.destination = valid URI;
controller.analytics.source = valid URI;

계정

사용account속성을 사용하여 Brightcove Analytics에서 계정 ID 값을 설정합니다. 이 속성은 다음이 없는 경우에만 유용합니다. accountId비디오 메타데이터의 값(예: 앱에서 Video Cloud 비디오를 혼합하여 사용하는 경우accountId속성 및 Video Cloud가 아닌 비디오(다음을 포함하지 않음) accountId재산).

controller.analytics.account = @"your accountId";

계정 속성을 설정하면 이 속성의 값이 아직 계정이 없는 동영상에 사용됩니다. accountId재산. 이 속성의 가장 유용한 값은 자신의 계정 ID입니다. 이러한 동영상은 분석 모듈에서 다음과 같이 보고됩니다. 기타/제3자 Brightcove Analytics는 현재 Brightcove 카탈로그 서비스를 통해 검색되지 않은 비디오에 대한 자세한 보고를 지원하지 않기 때문입니다.

Video Cloud 비디오의 경우 이 계정 재정의를 설정해도 보고에 영향을 주지 않습니다. accountId메타데이터 값은 변경되지 않습니다. 이 계정 값을 설정하여 해당 값을 무시할 수 없습니다.

목적지

사용destination이벤트가 발생한 위치의 값을 설정하는 속성입니다. 대상 속성의 값은 다음 예와 유사한 적절한 URI 형식이어야 합니다(MUST).

bcsdk://com.mhogben.source

대상 속성의 기본값은bcsdk://앱의 번들 식별자 다음에 앱의 info.plist파일.

예를 들어 iOS 샘플 앱에서 기본 대상 값은bcsdk://com.brightcove.sampleAppMainScreen.Destination다음과 같이 재정의할 수 있습니다.

controller.analytics.destination =
    @"myapp://com.mycompany.homescreen.destination";

인스턴스화하는 앱의 모든 화면BCOVPlaybackController앱 자체 내에서 동영상 조회수를 더 잘 추적하기 위해 이 대상 재정의를 고유한 값으로 설정할 수 있습니다.

원천

사용source속성을 사용하여 최종 사용자를 대상 URI로 보낸 URI를 설정합니다. 대상 값과 마찬가지로 소스 속성의 값은 다음 예와 유사한 적절한 URI 형식이어야 합니다(MUST).

bcsdk://com.mhogben.source

소스 속성의 기본값은 nil입니다. 값이 nil이면 소스 속성이 Analytics 서버로 전송되지 않습니다.

다음은 소스 속성을 설정하는 방법의 예입니다.

controller.analytics.source =
    @"myapp://com.mycompany.homescreen.destination.source";
대상 및 소스 필드에서 사용하기 위한 URI 구조에 대한 자세한 내용은 다음을 참조하십시오. URI 체계위키피디아 페이지. 다음과 같은 도구를 사용할 수 있습니다. URL 인코더문자를 UTF-8 인코딩 형식으로 변환합니다.
