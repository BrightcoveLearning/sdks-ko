Xcode 프로젝트 설정

앱에 대한 새 Xcode 프로젝트를 만듭니다.

  1. Xcode에서 새 iOS 프로젝트 생성을 시작합니다. 선택하다새 Xcode 프로젝트 생성 . 또는 Xcode 상단 메뉴에서 다음을 선택할 수 있습니다. 파일 , 새로운 , 프로젝트 .

  2. 선택하다단일 보기 응용 프로그램템플릿을 선택하고다음 .

  3. 프로젝트 정보를 다음과 같이 설정합니다.

    • 상품명: 단순 비디오 재생
    • 팀:없음

      팀 필드는 선택 사항입니다. 앱 스토어에서 앱을 배포하려면 Apple 개발자 프로그램을 통해 소속된 팀을 선택해야 합니다. 코드 서명에는 팀 이름이 필요합니다. 이 예에서는 없음을 선택합니다.

    • 조직 이름:당신의 회사 이름

      이렇게 하면 앱 스토어에서 제품 이름이 고유해집니다.

  4. 프로젝트를 저장하고 생성할 위치를 선택합니다.

  5. 이제 프로젝트를 닫습니다( 예, 닫습니다. 이것은 중요합니다! )

프로젝트에 SDK 및 해당 종속성 추가

SDK 및 해당 종속성을 프로젝트에 추가하는 가장 쉬운 방법은 CocoaPods를 사용하는 것입니다.

CocoaPods는 프로젝트에 라이브러리를 추가하는 종속성 관리자입니다. 필수는 아니지만 설치가 더 쉽습니다. CocoaPods를 설치하려면 설명서의 지침을 참조하십시오. 코코아팟대지.

  • CocoaPods 버전 1.0 이상이 필요합니다.
  • CocoaPods( pod도구) 기존 Xcode 프로젝트에서 프로젝트 파일을 수정합니다. 이 작업을 수행하기 전에 백업을 만들 수 있습니다.

  1. 프로젝트 폴더에서 이라는 일반 텍스트 파일을 만듭니다. 팟파일 (파일 확장자 없음).

  2. 텍스트 편집기를 사용하여 다음 코드 줄을 Podfile에 추가하고 저장합니다. 이 코드는 다음을 수행합니다.

    • 1행: Brightcove 포드 사양에 대한 GitHub 위치를 가리킵니다.
    • 3행: 정적 라이브러리 대신 프레임워크를 사용하도록 Pod를 설정합니다.
    • 5행: iOS 플랫폼 버전을 정의합니다.

    • 7-9행: Brightcove 네이티브 플레이어 SDK를 설치합니다.

      1. 에서 현재 네이티브 SDK 버전을 가져옵니다. iOS용 Brightcove 기본 SDK 개요문서.
      2. 값 바꾸기 '6.3.11'현재 SDK 버전 번호로.

      3. 종속성 관리에 대한 자세한 내용은 다음을 참조하십시오. CocoaPods로 종속성을 관리하는 방법기사.

       
      NS target속성은 CocoaPods 버전 1.0 이상에 사용됩니다.
    source 'https://github.com/brightcove/BrightcoveSpecs.git'

use_frameworks!

platform :ios, '11.0' # (or whatever version you want)

target 'Simple-Video-Playback' do
  pod 'Brightcove-Player-Core', '6.3.11'
end

  3. 터미널 세션을 열고단순 비디오 재생 Xcode 프로젝트 폴더.

  4. 터미널 세션에서 다음 명령을 입력합니다.

    pod install

    그리고 누르다반품실행합니다. Brightcove Player SDK가 프로젝트에 추가되었음을 나타내는 일련의 메시지가 터미널에 표시되어야 합니다.

  5. 마지막 줄을 주목하라 , 중요합니다. 이 시점부터 Simple-Video-Playback.xcworkspace Xcode의 파일, ~ 아니다 NS Simple-Video-Playback.xcodeproj 파일. NS pod install명령이 이것을 생성합니다.xcworkspace프로젝트에 대한 파일입니다.

  • 정기적으로 Podfile을 만들고 편집하는 경우 다음을 사용해 볼 수 있습니다. 코코아팟 앱이전 단계를 완료합니다.
  • CocoaPods를 사용하지 않으려면 다음을 수행할 수 있습니다. 최신 버전의 SDK 다운로드수동으로 프로젝트에 추가하십시오.

앱의 오디오 동작 설정

오디오 세션은 앱 수준에서 오디오 동작을 처리합니다. 에 대해 자세히 알아보기 AV오디오세션수업.

여러 오디오 세션 범주 및 설정에서 선택하여 앱의 오디오 동작을 사용자 지정할 수 있습니다. 최고의 오디오 세션 카테고리 선택당신의 앱을 위해.

이 샘플에서는 다음을 사용합니다. AVAudioSessionCategoryPlayback . 화면이 잠겨 있고 벨소리/무음 스위치가 무음으로 설정된 경우에도 오디오를 재생합니다. 간단하게 유지하기 위해 이 코드를 App Delegate에 넣습니다.

  1. 프로젝트에서 App Delegate 구현 파일( AppDelegate.m ).

  2. 에서didFinishLaunchingWithOptions메서드에서 오디오 세션 범주를 설정하는 코드를 추가합니다. 수입했는지 확인AVFoundation뼈대.

    • 오디오가 예상할 때 재생되도록 하려면 아래 코드가 필요합니다. 예를 들어 이 코드를 설정하지 않으면 음소거 스위치가 켜져 있을 때 비디오가 들리지 않습니다.
    • 샘플의 단순화를 위해 이것을 앱 대리자에 넣습니다. 자신의 앱에서 이것을 사용하는 방법에 대한 자세한 내용은오디오 세션 프로그래밍 가이드 .

  [super viewDidLoad];
  // Do any additional setup after loading the view, typically from a nib.

  // Set up our player view. Create with a standard VOD layout.
  BCOVPUIPlayerView *playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController:self.playbackController options:nil controlsView:[BCOVPUIBasicControlView basicControlViewWithVODLayout] ];

  playerView.translatesAutoresizingMaskIntoConstraints = NO;
  [NSLayoutConstraint activateConstraints:@[
      [playerView.topAnchor constraintEqualToAnchor:_videoContainer.topAnchor],
      [playerView.rightAnchor constraintEqualToAnchor:_videoContainer.rightAnchor],
      [playerView.leftAnchor constraintEqualToAnchor:_videoContainer.leftAnchor],
      [playerView.bottomAnchor constraintEqualToAnchor:_videoContainer.bottomAnchor],
  ]];
  _playerView = playerView;

  // Associate the playerView with the playback controller.
  _playerView.playbackController = _playbackController;

  [self requestContentFromPlaybackService];
}

  1. 아래viewDidLoad함수, 이름이 지정된 함수 생성requestContentFromPlaybackService다음과 같이:

    - (void)requestContentFromPlaybackService
{
[self.playbackService findVideoWithVideoID:kViewControllerVideoID parameters:nil completion:^(BCOVVideo *video, NSDictionary *jsonResponse, NSError *error) {

	if (video)
	{
		[self.playbackController setVideos:@[ video ]];
	}
	else
	{
		NSLog(@"ViewController Debug - Error retrieving video: `%@`", error);
	}

}];

미디어 보안 관리

ATS(앱 전송 보안)는 앱과 웹 서비스 간에 보안 연결을 적용합니다. Apple의 iOS 9 SDK 릴리스와 함께앱 전송 보안(ATS)추가되었습니다.

  1. 다음 상황 중 하나가 앱에 적용됩니다.

    • 기본적으로 iOS용 Brightcove 기본 SDK는 소스 선택 정책을 사용하여 HTTP 소스보다 HTTPS를 선택하므로 ATS가 활성화된 앱을 빌드할 수 있습니다.

      이제 앱을 실행할 준비가 되었습니다.

       

    • 원격 자산이 HTTP 소스 대신 HTTPS를 사용하는 경우 ATS가 활성화된 앱을 빌드할 수 있습니다.

      이제 앱을 실행할 준비가 되었습니다.

       

    • HTTP 소스를 사용 중이거나 앱에 다른 HTTP 호출이 있는 경우 다음 오류 메시지가 표시될 수 있습니다.

      App Transport Security has blocked a cleartext HTTP (http://) resource load since it is insecure.
Temporary exceptions can be configured via your app's Info.plist file.

      이는 ATS가 활성화되었지만 시스템이 ATS 요구 사항을 충족하도록 구성되지 않았음을 의미합니다. 이 상황을 해결하려면 다음을 참조하십시오. 앱 전송 보안(ATS) 작업문서.
