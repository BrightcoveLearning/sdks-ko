DRM 솔루션

DRM(디지털 권한 관리)은 비디오 데이터를 암호화하고 라이선스 정책에 따라 잠금을 해제하여 비디오 콘텐츠를 보호합니다. DRM에 대한 Video Cloud 지원에 대한 자세한 내용은 다음을 참조하십시오. 개요: 비디오 클라우드의 디지털 권한 관리(DRM) .

Brightcove Native Player SDK 플랫폼은 다음 DRM 솔루션을 지원합니다.

플랫폼 DRM 솔루션
iOS용 네이티브 플레이어 SDK FairPlay 스트리밍이 포함된 HLS
tvOS용 네이티브 플레이어 SDK FairPlay 스트리밍이 포함된 HLS
Android용 네이티브 플레이어 SDK Widevine 모듈형 DASH

비디오 클라우드 고객

다음 단계에서는 DRM 지원 콘텐츠를 만듭니다.

Apple FairPlay 스트리밍 사용

iOS 앱에서 Widevine Classic을 사용하는 경우 다음과 같이 Apple FairPlay 스트리밍 사용으로 전환할 수 있습니다.

  1. 계정이 DRM을 지원하지 않는 경우 계정 관리자에게 문의하여 DRM을 활성화하십시오.

    DRM 암호화를 사용하려면 계정에서 이러한 기능을 활성화해야 합니다. 자세한 내용은 계정 관리자에게 문의하세요.

  2. 에 대해 활성화된 계정으로동적 전달및 DRM을 사용하면 비디오를 수집할 때 DRM용으로 비디오가 자동으로 패키징됩니다. DRM은 Dynamic Delivery 자산에 소급 적용되지 않기 때문에 오래된 콘텐츠를 다시 코드 변환해야 합니다(당사의 엔지니어링 팀이 기존 콘텐츠에 DRM을 적용하는 데 도움을 줄 수 있음).

  3. FairPlay로 보호된 비디오에 대한 지원은 iOS용 Brightcove Player SDK의 핵심 프레임워크에 통합되었습니다. 자세한 내용은 다음 문서를 참조하십시오.

    • Brightcove FairPlay 플러그인에서 마이그레이션
    • FairPlay를 사용한 기본 플레이어 샘플
     
    외부 화면에 FairPlay 콘텐츠를 표시하려면 iOS 코드 조각문서.

Google Widevine 모듈 사용

Android 앱에서 Widevine Classic을 사용하는 경우 다음과 같이 Google Widevine Modular 사용으로 전환할 수 있습니다.

  1. 계정이 DRM을 지원하지 않는 경우 계정 관리자에게 문의하여 DRM을 활성화하십시오.

    DRM 암호화를 사용하려면 계정에서 이러한 기능을 활성화해야 합니다. 자세한 내용은 계정 관리자에게 문의하세요.

  2. 에 대해 활성화된 계정으로동적 전달및 DRM을 사용하면 비디오를 수집할 때 DRM용으로 비디오가 자동으로 패키징됩니다. DRM은 Dynamic Delivery 자산에 소급 적용되지 않기 때문에 오래된 콘텐츠를 다시 코드 변환해야 합니다(당사의 엔지니어링 팀이 기존 콘텐츠에 DRM을 적용하는 데 도움을 줄 수 있음).

  3. 이제 앱에서 Android용 Brightcove 네이티브 SDK를 사용할 준비가 되었습니다. Widevine Modular 콘텐츠를 보려면 ExoPlayer 비디오 보기를 사용해야 합니다. 자세한 내용은 다음 문서를 참조하십시오.

    • ExoPlayer를 사용한 DRM 보호 콘텐츠
    • Widevine Modular를 사용하는 기본 플레이어 샘플
     
    ExoPlayer 동영상 보기( BrightcoveExoPlayerVideoView )는 추가 플러그인 없이 Widevine Modular를 지원합니다.

브라이트코브 플레이어 고객

Apple FairPlay 스트리밍 사용

iOS 앱에서 Widevine Classic을 사용하는 경우 다음과 같이 Apple FairPlay 스트리밍 사용으로 전환할 수 있습니다.

  1. FairPlay의 경우 분할 및 암호화된 비디오와 함께 HLS 매니페스트를 사용하는 DRM 보호 콘텐츠를 생성해야 합니다.

  2. Brightcove FairPlay 라이선스 서버를 사용하지 않을 것이기 때문에 FairPlay 라이선스와 관련된 일부 속성을 정의해야 합니다.

    FairPlay로 보호된 비디오에 대한 지원은 iOS용 Brightcove Player SDK의 핵심 프레임워크에 통합되었습니다. 자세한 내용은 다음 문서를 참조하십시오.

    • Brightcove FairPlay 플러그인에서 마이그레이션
    • FairPlay를 사용한 기본 플레이어 샘플
     
    외부 화면에 FairPlay 콘텐츠를 표시하려면 iOS 코드 조각문서.

Google Widevine 모듈 사용

Android 앱에서 Widevine Classic을 사용하는 경우 다음과 같이 Google Widevine Modular 사용으로 전환할 수 있습니다.

  1. Widevine Modular의 경우 분할 및 암호화된 비디오와 함께 MPEG-DASH 매니페스트를 사용하는 DRM 보호 콘텐츠를 생성해야 합니다.

  2. Android용 Brightcove 기본 SDK는 기본적으로 Brightcove의 Widevine 라이선스 서버를 사용하도록 구성되어 있습니다. Widevine 라이선스 서버를 사용하도록 동영상을 구성해야 합니다. 다음은 예입니다.

      import com.brightcove.player.display.WidevineMediaDrmCallback;
  import com.brightcove.player.media.DeliveryType;
  import com.brightcove.player.model.Video;
  String performVideoUrl = "http://url.to.your.video";
  String performLicenseUrl = "https://url.to.your.licensing.server";
  Video performVideo = Video.createVideo(performVideoUrl, DeliveryType.DASH);
  performVideo.getProperties().put(WidevineMediaDrmCallback.DEFAULT_URL, performLicenseUrl);

  3. 이제 앱에서 Android용 Brightcove 네이티브 SDK를 사용할 준비가 되었습니다. Widevine Modular 콘텐츠를 보려면 ExoPlayer 동영상 보기를 사용해야 합니다. 자세한 내용은 다음을 참조하십시오.

    • ExoPlayer를 사용한 DRM 보호 콘텐츠
    • Widevine Modular를 사용하는 기본 플레이어 샘플
     
    ExoPlayer 동영상 보기( BrightcoveExoPlayerVideoView )는 추가 플러그인 없이 Widevine Modular를 지원합니다.
