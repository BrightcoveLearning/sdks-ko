개요

자막은 액세스하려는 개인을 위해 비디오에 대한 추가 정보를 제공합니다. 청각 장애인이 동영상의 오디오 부분에 액세스하려면 자막이 필요합니다. 캡션을 처음 사용하는 경우 다음을 참조하십시오. 개요: 비디오에 캡션 추가문서.

각 비디오 형식에 가장 적합한 캡션 유형을 이해하는 것이 중요합니다. 검토하여 시작비디오 형식 및 캡션문서. Brightcove는 WebVTT 캡션과 함께 HLS 또는 DASH 비디오를 사용할 것을 권장합니다.

당신이 사용할 때동적 수집 API Dynamic Delivery 비디오에 SRT 또는 DFXP 캡션을 추가하려면 캡션이 WebVTT 형식으로 자동 변환됩니다. 에 대해 자세히 알아보기 Dynamic Ingest API로 텍스트 트랙 추가 .
Android용 기본 SDK는 WebVTT Ruby 태그( <ruby>그리고<rt> ).

캡션

HLS 및 DASH 비디오에는 캡션 재생을 위한 두 가지 옵션이 있습니다.

  • 매니페스트 내 WebVTT
  • 외부(사이드카) WebVTT

매니페스트 내 WebVTT

HLS 또는 DASH 비디오와 함께 매니페스트 내 WebVTT 캡션을 사용하면 캡션이 작동합니다. 이러한 캡션을 구현하기 위해 추가로 수행해야 하는 작업은 없습니다.

매니페스트 내 WebVTT 캡션은 비디오와 함께 수집됩니다. Dynamic Delivery와 함께 Dynamic Ingest를 사용하는 경우에만 매니페스트 내 파일을 생성할 수 있습니다. 자세한 내용은 다음을 참조하세요.

  • WebVTT 파일 수집
  • Dynamic Ingest API를 사용하여 캡션 수집

캡션은 다음과 같은 경우 매니페스트 내로 인식됩니다. m3u8파일에는 다음 태그가 포함되어 있습니다.

#EXT-X-MEDIA:TYPE=SUBTITLES

예시

대본 텍스트 트랙의 위치 텍스트 트랙 유형 #EXT-X-MEDIA:TYPE=SUBTITLES 태그
Dynamic Delivery 인제스트된 비디오
동적 수집 API		 CDN 분할 예

노트:
  • Studio의 미디어 모듈을 사용하여 비디오에 캡션을 추가하면 매니페스트 내 캡션이 생성되지 않습니다.
  • 레거시 수집을 사용하면 매니페스트 내 캡션이 생성되지 않습니다.

외부(사이드카) WebVTT

외부 WebVTT 캡션은 동영상과 별도의 텍스트 파일입니다. 다음과 같이 캡션 파일을 비디오와 연결할 수 있습니다.

  • 미디어 모듈을 사용하여 캡션 추가
  • CMS API를 사용하여 캡션 추가
