개요

교차 기기 재개를 통해 시청자는 한 기기에서 동영상 시청을 시작하고 나중에 다른 기기에서 시청을 중단한 부분부터 계속 시청할 수 있습니다.

이 기능에 익숙하지 않은 경우 다음을 참조하십시오. 개요: 교차 기기 이력서문서.

요구 사항

Brightcove 네이티브 SDK를 사용한 교차 장치 이력서에는 다음 요구 사항이 필요합니다.

브라이트코브 네이티브 SDK 버전

  • Android 6.13.1 이상용 네이티브 SDK
  • iOS 6.0.1 이상용 네이티브 SDK, Pulse 6.7.5 이상용, SSAI 6.7.7 이상용

기기 OS 버전

  • 안드로이드 6.0 이상
  • iOS 11.0 이상

설정

이 기능은 Brightcove Video Cloud 계정이 있는 모든 사용자가 사용할 수 있습니다.

시작하려면 다음을 수행하십시오.

  • 계정 관리자에게 문의하여 교차 기기 재개용 계정을 활성화하십시오.
  • 사용 중인 비디오가 Dynamic Delivery를 위해 수집되었는지 확인하십시오.

교차 기기 이력서 구현

교차 기기 이력서를 구현하려면 다음 단계를 따르세요.

  1. Brightcove 분석을 위한 사용자 식별자 설정

    뷰어 데이터가 자동으로 Brightcove 분석으로 전송되더라도 사용자 식별자를 설정해야 합니다.

    인증 게이트웨이 또는 일부 ID 관리 솔루션을 사용하여 뷰어를 추적할 수 있습니다. 이 뷰어 ID를 사용자 식별자로 사용하여 Brightcove 분석에 전달하십시오.

  2. Cross-Device Resume(XDR) API에서 뷰어 재생 위치 가져오기

  3. 재생 재개

안드로이드 구현

Android용 네이티브 SDK의 경우 다음 단계를 따르세요.

  1. Brightcove 분석에 대한 사용자 식별자를 설정합니다. 

    HashMap<String, String> baseParams = new HashMap<>();
baseParams.put(Analytics.Fields.USER, "viewer id");
baseParams.put(Analytics.Fields.APPLICATION_ID, "application id");

HashMap<String, Object> eventParams = new HashMap<>();
eventParams.put(Analytics.Fields.BASE_PARAMS, baseParams);
eventEmitter.emit(EventType.ADD_ANALYTICS_BASE_PARAMS, eventParams);

  2. XDR(Cross-Device Resume) API에서 뷰어 재생 위치를 가져옵니다.

    자세한 내용은 다음을 참조하세요.

    • XDR(Cross-Device Resume) API에서 재생 위치 가져오기
    • 교차 기기 이력서(XDR) API 참조

  3. 재생을 재개합니다.

    XDR API에서 뷰어 재생 위치를 가져오면 해당 지점에서 재생을 재개할 수 있습니다. 를 사용하여 재생 헤드 위치를 설정합니다. brightcoveVideoView.seekTo()방법. 

    final VideoPlaybackController playbackController = brightcoveVideoView.getPlaybackController();
eventEmitter.on(EventType.VIDEO_DURATION_CHANGED, new EventListener() {
  @Override
  public void processEvent(final Event event) {
    playbackController.setAdsDisabled(true);
    brightcoveVideoView.seekTo(<viewer playhead position>);
  }
});

eventEmitter.on(EventType.DID_SEEK_TO, new EventListener() {
  @Override
  public void processEvent(final Event event) {
    playbackController.setAdsDisabled(false);
  }
});

iOS 구현

iOS용 네이티브 SDK의 경우 다음 단계를 따르세요.

  1. Brightcove 분석에 대한 사용자 식별자를 설정합니다. 

    @try
  {
    [BCOVGlobalConfiguration.sharedConfig setValue:@{
      @"privateUser": self.viewer id,
      @"privateApplication": @"application id"
    }
    forKey:@"privateSessionAnalytics"];
  }
  @catch (NSException *e)
  {
    NSLog(@"%@", e.description);
  }

  2. XDR(Cross-Device Resume) API에서 뷰어 재생 위치를 가져옵니다.

    자세한 내용은 다음을 참조하세요.

    • XDR(Cross-Device Resume) API에서 재생 위치 가져오기
    • 교차 기기 이력서(XDR) API 참조

  3. 재생을 재개합니다.

    XDR API에서 뷰어 재생 위치를 가져오면 해당 지점에서 재생을 재개할 수 있습니다. 다음 방법을 사용하여 재생 헤드 위치를 설정합니다. 

    -[BCOVPlaybackController seekWithoutAds:(CMTime)seekToTime completionHandler:(void (^)(BOOL finished))completion]

    이를 통해 시청자가 이전에 예약된 광고를 시청하지 않고도 특정 시간에 재생을 재개할 수 있습니다. seekToTime .

    에 대비하여seekWithoutAds:completionHandler: , 장애를 입히다autoPlay설정할 때BCOVPlaybackController .

    Apple은 다음 상태를 기다릴 것을 권장합니다. AVPlayerItem사용하기 전에 바로 플레이 가능으로 변경하려면AVPlayerItem . 따라서 전화seekWithoutAds:completionHandler:에서kBCOVPlaybackSessionLifecycleEventReady핸들러playbackController:playbackSession:didReceiveLifecycleEvent당신의 방법BCOVPlaybackControllerDelegate .
