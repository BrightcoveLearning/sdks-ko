개요

이제 Brightcove Native SDK는측정 SDK 열기 (OM SDK). Google IMA와 통합된 OM SDK를 사용하면 광고 측정 및 확인을 위해 타사 도구를 사용할 수 있습니다.

Open Measurement SDK는 광고 미디어 요소를 오버레이하는 모든 보기를 장애물로 간주하여 광고 조회 가능성을 줄입니다. 비디오 컨트롤은 "친숙한" 장애물로 간주될 수 있지만 광고 조회가능성 측정에서 제외하려면 OM SDK에 등록해야 합니다.

iOS

iOS용 Brightcove 네이티브 SDK는 OM SDK 사용을 지원합니다. 타사 광고 측정 구현에 대한 자세한 내용은 iOS용 IMA SDK에서 측정 열기문서.

UI 오버레이

IMA 광고 보기를 오버레이하는 UI 컨트롤 요소를 사용하는 경우 이를 OM SDK에 등록하는 것이 좋습니다. 자세한 내용은광고 오버레이 등록 iOS용 네이티브 SDK 참조 섹션.

IMA 설정

기본적으로 iOS용 Brightcove 네이티브 SDK는playerType그리고playerVersion다음 값을 가진 속성:

imaSettings.playerType = @"bcov/ios-sdk-player";
imaSettings.playerVersion = [BCOVIMAComponent versionIdentifier];

이러한 값을 재정의할 수 있습니다. 한 가지 예는 Google IMA에서 직접 보고되는 광고 측정항목을 얻는 것입니다. 자신의 값을 사용하려면 다음을 설정하십시오. imaSettings물체:

imaSettings.playerType = @"your-player-type";
imaSettings.playerVersion = @"your-player-version"
각 속성의 최대 길이는 20자입니다.

iOS용 네이티브 SDK와 함께 IMA 광고를 사용하는 방법에 대한 코드 세부정보는기본IMAPlayer 샘플 .

기계적 인조 인간

Android용 Brightcove 네이티브 SDK는 OM SDK 사용을 지원합니다. 타사 광고 측정 구현에 대한 자세한 내용은 Android용 IMA SDK에서 측정 열기문서.

UI 오버레이

IMA 광고 보기를 오버레이하는 UI 컨트롤 요소를 사용하는 경우 이를 OM SDK에 등록하는 것이 좋습니다. 이렇게 하려면 코드가 다음과 같이 보일 것입니다.

public class MainActivity extends BrightcovePlayer {
 // ...
 private AdDisplayContainer adDisplayContainer;
 private View adOverlayView;
 // ...

 @Override
 protected void onCreate(Bundle savedInstanceState) {
   // ...
   adOverlayView = findViewById(R.id.ad_overlay);
   setupGoogleIMA();
   // ...
 }

 private void setupGoogleIMA() {
   // Show adOverlayView
   eventEmitter.on(EventType.AD_BREAK_STARTED, (event) ->  {
       adOverlayView.setVisibility(View.VISIBLE);
   });

   // Hide adOverlayView
   eventEmitter.on(EventType.AD_BREAK_COMPLETED, (event) ->  {
       adOverlayView.setVisibility(View.GONE);
   });

   // Create ImaSdkFactory
   final ImaSdkFactory sdkFactory = ImaSdkFactory.getInstance();
   eventEmitter.on(GoogleIMAEventType.ADS_REQUEST_FOR_VIDEO, new EventListener() {
     @Override
     public void processEvent(Event event) {
       // Create a container object to register the friendly obstructions.
       adDisplayContainer = sdkFactory.createAdDisplayContainer();
       adDisplayContainer.registerVideoControlsOverlay(adOverlayView);
       // ...
     }
   });

   eventEmitter.on(EventType.COMPLETED, (event) -> {
     if (adDisplayContainer != null) {
       // Unregister the friendly obstructions
       adDisplayContainer.unregisterAllVideoControlsOverlays();
     }
   });
 }
}

IMA 설정

기본적으로 iOS용 Brightcove 네이티브 SDK는playerType그리고playerVersion다음 값을 가진 속성:

imaSdkSettings.setPlayerType("bcov/and-sdk-player");
imaSdkSettings.setPlayerVersion("<replaced with current sdk version>");

이러한 값을 재정의할 수 있습니다. 한 가지 예는 Google IMA에서 직접 보고되는 광고 측정항목을 얻는 것입니다. 자신의 값을 사용하려면 다음을 설정하십시오. imaSdkSettings물체:

ImaSdkSettings imaSdkSettings = imaSdkFactory.createImaSdkSettings();
imaSdkSettings.setPlayerType("your-player-type");
imaSdkSettings.setPlayerVersion("your-player-version");
GoogleIMAComponent googleIMAComponent = new GoogleIMAComponent(brightcoveVideoView, eventEmitter, imaSdkSettings);
각 속성의 최대 길이는 20자입니다.
