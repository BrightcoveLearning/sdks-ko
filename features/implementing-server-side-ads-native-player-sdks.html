    네이티브 SDK로 서버 측 광고 구현

    이 항목에서는 Dynamic Delivery를 위해 수집 된 비디오에서 Brightcove 네이티브 플레이어 SDK를 사용하여 서버 측 광고를 재생하는 방법을 알아 봅니다.

    개요

    서버 측 광고 삽입 (SSAI를) 사용하면 동영상에 광고를 삽입하여 브라우저의 광고 차단기에 의해 차단되지 않도록 할 수 있습니다. Dynamic Delivery는 스토리지 공간을 줄이고 미디어를 동적으로 패키징하는 차세대 수집 및 전달 시스템입니다. 사용에 대해 더 알아보기동적 전달당신의 비디오.

    SSAI는 DRM 및 비 DRM 콘텐츠 모두에서 작동합니다.

    참고:
    • SSAI를 사용하려면 비디오 클라우드 계정을동적 전송으로구성하고 SSAI에 대해 활성화해야 합니다. 이 기능을 사용하려면 계정 관리자에게 문의하십시오.
    • SSAI는 VOD (주문형 비디오) 와 함께 작동하며 라이브 재생을 지원합니다.
    • 제한 사항 목록은SSAI의 개요문서를 참조하십시오.

    동적 전송으로 수집된 Video Cloud에서 검색한 모든 동영상에는 광고 구성에 VMAP 파일에 지정된 광고가 포함됩니다.

    Video Cloud에 저장된 비디오 콘텐츠로 서버 측 광고를 재생하려면 다음 단계를 따르십시오.

    1. 광고 구성 만들기

    2. 앱 빌드:

      • 안드로이드 구현
      • iOS 구현
      • tvOS 구현
    3. 광고가있는 동영상 재생

    광고 구성 만들기

    광고 구성은 VMAP (Video Multiple Ad Playlist) 파일, 비콘 및 기타 구성에 대한 URL을 포함하여 SSAI 재생의 다양한 측면을 정의합니다. 광고 구성을 만들려면 다음 단계를 따르세요.

    1. Video Cloud Studio에서관리자메뉴 및 선택서버 측 광고 설정 .

    2. 광고 구성에 대한 정보를 추가하고저장을선택합니다.

    자세한 내용은서버 측 광고 설정 구성문서.

    SSAI API를 사용하여 광고 구성을 만들려면 Video Cloud SSAI 광고 구성 API문서.

    안드로이드 구현

    앱에서 다음 단계에 따라 광고 구성을 가져오고 동영상을 재생하세요.

    1. Android Studio에서 MainActivity.java파일.
    2. 다음 import 문을 추가합니다. 
      import com.brightcove.ssai.SSAIComponent;
import com.brightcove.player.network.HttpRequestConfig;
    3. 계정 ID, 정책 키, 동영상 ID 및 광고 구성 ID에 대한 상수를 정의합니다. 
      private String accountId = "your account id";
private String policyKey = "your policy key";
private String videoId = "your video id";
private String adConfigId = "your ad configuration id";
      에서 정책 키를 얻거나 생성하는 방법을 알아보십시오. 정책 키문서.
    4. Playback API에서 데이터를 검색하기위한 비동기 메서드를 제공하는 카탈로그 서비스의 인스턴스를 만듭니다. SSAI 구성 요소의 인스턴스를 만듭니다. 
      Catalog catalog = new Catalog(eventEmitter, myAccountId, myPolicyKey);
SSAIComponent plugin = new SSAIComponent(appContext, getBrightcoveVideoView());
    5. adConfigId를 URL 매개 변수로 설정합니다. 
      HttpRequestConfig httpRequestConfig = new HttpRequestConfig.Builder()
  .addQueryParameter("ad_config_id", myAdConfig)
  .build();
    6. 카탈로그 서비스를 호출하여 Playback API에서 광고 구성과 함께 동영상을 검색합니다. 지정된 광고로 비디오 재생을 시작하십시오. 
      catalog.findVideoByID(myVideoId, httpRequestConfig, new VideoListener() {
    @Override
    public void onVideo(Video video) {

    // Process the Video
    plugin.processVideo(video);
  }
});

    자막

    Android 용 네이티브 SDK는 SSAI를 사용하는 매니페스트 캡션 만 지원합니다. 사이드카 (또는 원격) 캡션은 콘텐츠 및 광고 시간과 동기화되지 않을 수 있습니다.

    사내 행사 및 이벤트 소식을 비디오로 제공하십시오.

    앱에서 다음 단계에 따라 광고 구성을 가져오고 동영상을 재생하세요.

    1. Xcode에서 ViewController.m파일.
    2. iOS 용 SSAI 플러그인을 가져옵니다. 
      import BrightcoveSSAI
    3. 계정 ID, 정책 키, 동영상 ID 및 광고 구성 ID에 대한 상수를 정의합니다. 
      struct Constants {
  static let AccountID = "your account id"
  static let PlaybackServicePolicyKey = "your policy key"
  static let VideoId = "your video id"
  static let AdConfigId = "your ad configuration id"
}
      에서 정책 키를 얻거나 생성하는 방법을 알아보십시오. 정책 키문서.
    4. 정의BCOVPlaybackService Playback API에서 데이터를 검색하기위한 비동기 메서드를 제공하는 클래스입니다. 
      private lazy var playbackService: BCOVPlaybackService = {
  let factory = BCOVPlaybackServiceRequestFactory(accountId: Constants.AccountID, policyKey: Constants.PlaybackServicePolicyKey, baseURLStr: "https://edge.api.brightcove.com/playback/v1")
  return BCOVPlaybackService(requestFactory: factory)
}()
    5. 검색어 매개 변수 값을 광고 구성 ID로 설정합니다. 
      let queryParameters = [kBCOVPlaybackServiceParamaterKeyAdConfigId: Constants.AdConfigId]

    6. 을 부르다playbackService Playback API에서 광고 구성과 함께 동영상을 검색합니다. 지정된 광고로 비디오 재생을 시작하십시오.

      playbackService.findVideo(withVideoID: Constants.VideoId, parameters: queryParameters) { [weak self] (video: BCOVVideo?, jsonResponse: [AnyHashable: Any]?, error: Error?) -> Void in

  guard let _video = video else {
    print("ViewController Debug - Error retrieving video: \(error?.localizedDescription ?? "unknown error")")
    return
  }

  self?.playbackController?.setVideos([_video] as NSFastEnumeration)
}
