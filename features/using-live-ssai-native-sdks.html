개요

서버측 광고 삽입( 싸이 )를 사용하면 비디오 콘텐츠 스트림에 광고를 연결하여 TV와 같은 시청 경험을 제공할 수 있습니다. 네이티브 SDK로 라이브 스트림을 재생하려면 비디오 요청과 함께 재생 토큰을 전달해야 합니다.

살다싸이다음을 수행할 수 있습니다.

  • 인코더에서 보낸 큐 포인트를 사용하여 광고를 삽입하거나라이브 API
  • 사용하지 않은 광고 시간을 채우기 위해 "슬레이트" 자산을 수집합니다.
  • 서버 측의 라이브 스트림에 스티칭되는 광고가 있는 광고 차단기를 피하세요.

단계

라이브 스트림을 재생하려면싸이 , 이 차례를 따라라:

  1. 라이브 광고 구성 만들기비디오 클라우드 스튜디오 사용
  2. 선택 과목: 슬레이트 자산 만들기 , 및/또는큐 포인트 및 광고 비콘 삽입
  3. 라이브 이벤트 만들기
  4. 재생 토큰 가져오기

  5. 앱 빌드:

    • 안드로이드 구현
    • iOS 구현
  6. 광고 시간 요청
원하는 경우 Live API를 사용하여 스트림을 만들고 광고 시간을 구성할 수 있습니다. 자세한 내용은라이브 API: 서버 측 광고 삽입(SSAI)문서.

라이브 이벤트 만들기

기본적으로 라이브 스트림은 Brightcove CDN을 사용합니다. 원하는 경우 자신이 선택한 CDN(BYO CDN 또는 "사용자 고유의 대역폭 가져오기")을 사용할 수 있습니다. 자체 CDN을 사용하려면 이 섹션과재생 토큰 가져오기부분.

라이브 이벤트를 만들려면 다음 단계를 따르세요.

  1. 참조라이브 모듈에서 서버 측 광고 구현문서.
  2. 확장고급 옵션부분.

  3. 선택하다 SSAI(서버 측 광고 삽입) 활성화를 선택하고슬레이트 채우기 .

  4. 선택 과목 : 자체 CDN을 사용하려면 다음 추가 단계를 포함하세요.

    • 에서고급 옵션섹션, 선택다중 CDN 구성 추가옵션.

    • 자신의 CDN에 대한 레이블과 URL을 입력합니다. 자세한 내용은콘텐츠 전달 옵션문서.

  5. 라이브 이벤트 구성을 마치면이벤트 생성 .

재생 토큰 가져오기

라이브 스트림에서 SSAI를 활성화하면 플레이어 내장 코드에adConfigId매개변수. 이 값은 앱에서 비디오 콘텐츠를 요청할 때 사용할 재생 토큰입니다.

재생 토큰을 얻으려면 다음 단계를 따르세요.

  1. Video Cloud Studio에서살다기준 치수.
  2. 왼쪽 탐색에서게시 및 포함 .

  3. 에서라이브 플레이어 옵션섹션, 확장광고 구성 선택옵션을 선택하고 이전 단계에서 생성한 라이브 광고 구성을 선택합니다.

  4. 복사표준 삽입 코드당신의 클립보드에.

  5. 임베드 코드에서 다음을 복사합니다. VideoId매개변수 값. 이 값은 이후 단계에서 사용합니다. 다음과 유사해야 합니다.

    1700044540369289748

  6. 임베드 코드에서 다음을 복사합니다. adConfigId매개변수 값. 이 값은 이후 단계에서 사용합니다. NS adConfigId값은 Video Cloud 라이브러리에서 콘텐츠를 요청할 때 전달된 재생 토큰입니다. 다음과 유사해야 합니다.

    live.t0vbpNIjTPW6IZ29QsyuWrmgjvQjEWe3u_wPmaMqnhAmE05DCF7aAa-6fDIkvryJctH1rADJnfXYANy7tDg4agQFx23WIvsXMmAjz1BiNl-S3rFZ9tRdzRWdo1E4FTLkET6XcNuNZ2acTnzmow
  7. 또한 계정에 대한 계정 ID와 정책 키가 필요합니다.

안드로이드 구현

라이브 스트림을 재생하려면 다음을 포함해야 합니다. adConfigId비디오 요청에 대한 매개변수 및 재생 토큰 값.

앱을 빌드하려면 다음 단계를 따르세요.

  1. 사용기본 SSAI 샘플 앱코드의 시작점으로 사용할 수 있습니다.
  2. 에서 res/values/strings.xml파일에서 다음을 자신의 값으로 바꿉니다.
    • 계정 ID
    • 정책 키
    • 동영상 ID( 재생 토큰 가져오기위 섹션)

  3. 에서메인 액티비티.자바파일, 교체ad_config_id라이브 모듈에서 복사한 재생 토큰(라이브 URL)이 있는 값재생 토큰 가져오기위 섹션.

    private final String AD_CONFIG_ID_QUERY_PARAM_VALUE = "live.t0vbpNIjTPW6IZ29QsyuWrmgjvQjEWe3u_wPmaMqnhAmE05DCF7aAa-6fDIkvryJctH1rADJnfXYANy7tDg4agQFx23WIvsXMmAjz1BiNl-S3rFZ9tRdzRWdo1E4FTLkET6XcNuNZ2acTnzmow";

  4. NS ad_config_id재생 토큰 값이 HTTP 요청에 쿼리 매개변수로 추가됩니다.

    HttpRequestConfig httpRequestConfig = new HttpRequestConfig.Builder()
  .addQueryParameter(AD_CONFIG_ID_QUERY_PARAM_KEY, AD_CONFIG_ID_QUERY_PARAM_VALUE)
  .build();

  5. 라이브 스트림 비디오 ID와 업데이트된 HTTP 요청을 사용하여 Playback API에 대한 카탈로그 호출을 수행합니다.

    catalog.findVideoByID(getString(R.string.video_id), httpRequestConfig, new VideoListener() {
   @Override
   public void onVideo(Video video) {
       plugin.processVideo(video);
   }
 });

  6. 앱에서 실시간 스트림을 재생할 준비가 되었습니다.

    Android용 네이티브 SDK와 함께 SSAI에 활성화된 라이브 스트림을 사용하는 방법에 대한 자세한 내용은 6.9.0의 SSAI 플러그인 변경 사항블로그 게시물.
