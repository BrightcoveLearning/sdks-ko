개요

썸네일 검색을 통해 사용자는 타임라인을 따라 재생 헤드를 드래그하고 관련 콘텐츠의 미리보기로 썸네일을 볼 수 있습니다. 이를 통해 사용자는 비디오 파일을 빠르게 탐색하고 관심 있는 콘텐츠를 찾을 수 있습니다.

이미지는 언제 생성되나요?

Brightcove는 요청 시 이미지를 동적으로 생성합니다. Dynamic Delivery로 수집된 모든 비디오에는 첫 번째 요청에서 생성된 이미지가 있습니다. 이렇게 하면 사용하지 않는 비디오에 대해 이미지가 생성되지 않습니다. 생성된 이미지는 다른 이미지와 마찬가지로 반복적으로 볼 수 있도록 로컬 및 CDN에 캐시됩니다.

비디오의 이미지 빈도는 얼마입니까?

1시간 미만 콘텐츠의 경우 5초마다, 1시간 미만 콘텐츠의 경우 10초마다 이미지가 생성됩니다.

요구 사항

Brightcove Native SDK로 썸네일을 찾으려면 다음 요구 사항이 필요합니다.

이 기능을 사용하려면 Playback API v2를 사용해야 합니다.

기기 OS 버전

  • 안드로이드 6.0 이상
  • iOS 11.0 이상

브라이트코브 네이티브 SDK 버전

  • Android 6.12.0 이상용 네이티브 SDK
  • iOS 6.7.4 이상용 네이티브 SDK

시작하기

이 기능은 Brightcove Video Cloud 계정이 있는 모든 사용자가 사용할 수 있습니다.

Dynamic Delivery를 위해 비디오가 수집되었는지 확인하십시오.

안드로이드 구현

Android용 네이티브 SDK에서 썸네일 검색을 사용하려면 다음 단계를 따르세요.

  1. 에서 build.gradle파일에 다음 Gradle 종속성을 추가합니다. 

    implementation "com.brightcove.player:android-thumbnail-plugin:6.13.0"

  2. 에서메인 액티비티.자바파일, 인스턴스 생성ThumbnailComponent . 

    ThumbnailComponent thumbnailComponent = new ThumbnailComponent(brightcoveVideoView);

  3. 미리보기 미디어 컨트롤러를 설정합니다. 

    thumbnailComponent.setupPreviewThumbnailController();
  4. 전체 코드 예제는 다음을 참조하십시오. ThumbnailScrubberSampleApp

SSAI에 대한 추가 단계

SSAI(서버 측 광고 삽입)를 사용하는 경우 추가 단계가 있습니다.

SSAI 플러그인과 함께 Thumbnail 플러그인을 사용하려면 콘텐츠의 스티칭된 광고로 인해 발생하는 오프셋을 고려해야 합니다.

  1. 귀하의메인 액티비티.자바파일.
  2. 들어봐SSAIEventType.AD_DATA_READY이벤트.
  3. SSAI 비디오의 타임라인 개체를 검색합니다.

  4. 인스턴스 생성SSAIThumbnailDocumentCreatorWrapper다음 매개변수와 함께:

    • NS ThumbnailDocumentCreator . 당신은 사용할 수 있습니다DefaultThumbnailDocumentCreator .
    • SSAI 비디오의 타임라인 개체입니다.
    • 설정ThumbnailDocumentCreator ~로ThumbnailPlugin .

  5. 다음은 코드 예입니다. 

    eventEmitter.once(AD_DATA_READY, event -> {
  Timeline timeline = event.getProperty(SSAIEvent.VMAP_TIMELINE, Timeline.class);
  ThumbnailDocumentCreator creator = new SSAIThumbnailDocumentCreatorWrapper(new   DefaultThumbnailDocumentCreator(), timeline);
  thumbnailComponent.setThumbnailDocumentCreator(creator);
});

iOS 구현

iOS용 네이티브 SDK에서 썸네일 검색을 사용하려면 다음 단계를 따르세요.

  1. 새로운BCOVPlaybackController재생 컨트롤러에 대한 썸네일 검색을 활성화하거나 비활성화하는 속성입니다. 자세한 내용은 iOS용 네이티브 SDK 참조 .

  2. 기본값은YES . 썸네일 검색을 비활성화하려면 값을 다음으로 설정하십시오. NO .

    /**
 * Enables or disables thumbnail seeking for this playback controller.
 *
 * Default value is YES
 */
@property (nonatomic, readwrite, assign) BOOL thumbnailScrubbingEnabled;

노트

다음 참고 사항은 기본 SDK를 사용한 축소판 검색에 적용됩니다.

  • 썸네일 검색은 오프라인 재생에서 지원되지 않습니다.
