개요

클라이언트 측 광고에는 클라이언트 장치에서 광고 서버로 요청하는 작업이 포함됩니다. 이로 인해 광고 동영상을 검색하는 동안 약간의 버퍼링 시간이 발생할 수 있습니다. 보다 TV와 같은 경험을 위해 다음을 사용할 수 있습니다. 서버 측 광고 삽입 .

클라이언트 측 광고를 사용하면 광고 큐 포인트가 감지되거나 VMAP 응답의 광고 타이밍에 도달할 때까지 콘텐츠 비디오가 재생됩니다. 구성된 광고 서버에 요청이 이루어지면 광고 세부정보로 응답합니다. 그런 다음 플레이어는 요청된 광고의 재생을 시작합니다.

Brightcove 네이티브 SDK는 다음 기술에 대한 플러그인을 제공합니다.

  • Google의 양방향 미디어 광고(IMA)
  • 프리휠

네이티브 SDK는 Google의 Ad Manager 및 DFP(DoubleClick for Publishers), Google 애드센스 네트워크를 사용할 때 생성되는 VMAP를 지원합니다.

광고 유형

동영상에 사용할 수 있는 광고에는 세 가지 주요 유형이 있습니다.

선형 광고 - 메인 콘텐츠 영상을 일시 정지하고 플레이어의 전체 프레임에 표시되는 광고

비선형 광고 - 메인 콘텐츠 영상 위에 오버레이 광고가 표시됩니다. 텍스트, 그래픽 이미지 또는 비디오 오버레이가 될 수 있습니다.

컴패니언 광고 - 동영상 플레이어 외부에 표시되는 광고

광고 표준

표준 XML 파일을 사용하여 플레이어에게 광고를 재생할 때와 재생할 광고를 알릴 수 있습니다. NS 인터랙티브 광고 협회(IAB)은(는) 동영상 플레이어와 광고 단위 사이에 다음과 같은 공통 인터페이스를 제공합니다.

동영상 광고 게재 템플릿(VAST) - 디지털 비디오 플레이어에 광고를 제공하기 위한 범용 XML 스키마입니다. 이 파일은 표시할 광고를 지정합니다.

동영상 다중 광고 재생 목록(VMAP) - 이 파일은 프리롤, 미드롤, 포스트롤 및 비선형 광고를 포함한 VAST 광고의 재생 목록을 설명합니다. 여기에서 광고 시간이 발생하는 시간과 각 시간에 재생할 광고를 지정할 수 있습니다.

동영상 플레이어 광고 게재 인터페이스 정의(VPAID) - 양방향 광고 경험을 제공하기 위한 공통 인터페이스를 구축합니다.

Brightcove 네이티브 SDK는 VPAID 광고를 지원하지 않습니다.

광고 용어를 처음 사용하는 경우 먼저 다음을 검토하십시오. 동영상 광고 학습 가이드 .

광고 태그 정의

광고 태그는 검색 및 재생될 광고를 정의합니다. 다음 형식 중 하나를 사용할 수 있습니다.

  • VAST - XML 스키마에 대한 자세한 내용은 IAB VAST 사양 .
  • VMAP - XML 템플릿에 대한 자세한 내용은 IAB VMAP 사양 .

광고 태그 확인

광고 태그가 유효하고 예상대로 작동하는지 확인하는 것이 좋습니다. 이렇게 하면 구현 프로세스 중에 다른 문제를 디버깅하는 데 도움이 됩니다.

Google은비디오 제품군 검사기광고 태그를 확인할 수 있습니다.

FreeWheel은광고 미리보기광고 서버 응답을 확인합니다.

큐 포인트 만들기

VMAP 광고 태그를 사용하는 경우 이 단계를 건너뛸 수 있습니다. VMAP은 광고의 재생 목록을 정의하고 재생 시간을 지정합니다. 참고 Google의 IMA 광고 규칙 VMAP을 사용합니다.

VAST 광고 태그를 사용하는 경우 플레이어에게 광고를 삽입할 시기를 알려주는 큐 포인트를 만들어야 합니다. 앱에서 생성해야 합니다. 다음은 이를 수행하는 방법에 대한 몇 가지 예입니다.

안드로이드 구현

Android용 기본 SDK는 다음 기술을 사용하여 VMAP, VAST, 서버 측 광고 규칙 및 컴패니언 광고를 지원합니다.

구글 IMA

Google IMA 광고 규칙을 사용하려면 다음 단계를 따르세요.

  1. 찾아 검토광고 규칙 IMA샘플 앱.
  2. 열기메인 액티비티.자바파일.

  3. 귀하의 가치로 프로젝트를 사용자 정의하십시오. Video Cloud 라이브러리에서 콘텐츠를 검색하려면 다음 값이 필요합니다.

    • 재생 API 정책 키
    • 계정 ID
    • 동영상 ID

  4. IMA 광고에 액세스하기 위한 값 포함:

    • 광고 규칙 URL
  5. NS setupGoogleIMA()방법은 Brightcove IMA 플러그인과의 연결을 설정합니다.
  6. 들어봐GoogleIMAEventType.ADS_REQUEST_FOR_VIDEO이벤트. 여기에서 광고 요청 개체를 만들고 광고 태그 URL을 가리킵니다. 이 경우 광고 규칙 VMAP 파일입니다.
  7. NS setupAdMarkers()방법은 선택 사항입니다. 이 메서드는 IMA AdManager를 호출하고 탐색 표시줄에 광고 마커를 생성하여 사용자가 광고가 언제 재생되는지 시각적으로 볼 수 있도록 합니다.

프리휠

이러한 광고를 사용하려면 FreeWheel AdManager 라이브러리를 구매해야 합니다.

FreeWheel 광고를 사용하려면 다음 단계를 따르십시오.

  1. 연락하다프리휠 AdManager.aar 라이브러리 아카이브를 구입하여 일반적으로 /home/ 디렉토리의 루트에 있는 libs/ 디렉토리에 설치합니다. Android용 기본 SDK 및 FreeWheel 샘플 앱에는 FreeWheel AdManager 라이브러리 버전 6.28.0이 필요합니다.
  2. 찾아 검토프리휠샘플 앱.
  3. 열기메인 액티비티.자바파일.
  4. 귀하의 가치로 프로젝트를 사용자 정의하십시오. 이 샘플은 원격 비디오 자산을 사용합니다.

  5. FreeWheel 광고에 액세스하기 위한 값 포함:

    • 광고 URL
    • 광고 네트워크 ID
    • 광고 프로필
  6. NS setupFreeWheel()방법은 Brightcove FreeWheel 플러그인과의 연결을 설정합니다.
  7. 광고 URL을 가리키도록 FreeWheel 컨트롤러를 구성합니다. freeWheelController.setAdURL("http://demo.v.fwmrm.net/");
  8. 이것은 나중에 FreeWheel을 인스턴스화하는 데 사용됩니다. AdRequestConfiguration수업.
  9. 들어봐FreeWheelEventType.WILL_SUBMIT_AD_REQUEST이벤트. 여기에서 FreeWheel 컨트롤러를 얻고 프리롤, 미드롤, 포스트롤 및 오버레이 광고 슬롯을 설정합니다.

Google IMA 광고를 사용하려면 다음 단계를 따르세요.

  1. 찾아 검토기본 IMA 플레이어샘플 앱.
  2. 열기뷰컨트롤러파일.

  3. 귀하의 가치로 프로젝트를 사용자 정의하십시오. Video Cloud 라이브러리에서 콘텐츠를 검색하려면 다음 값이 필요합니다.

    • 재생 API 정책 키
    • 계정 ID
    • 동영상 ID

  4. IMA 광고에 액세스하기 위한 값 포함:

    • 게시자 ID
    • 언어
    • 광고 태그 URL
  5. 생성playbackController광고 요청 정책(VAST, VMAP 또는 서버 측 광고 규칙)을 포함한 적절한 IMA 설정을 사용합니다.
  6. 광고 이벤트는 수명 주기 이벤트를 통해 BCOVIMA 플러그인에 의해 방출됩니다. 이러한 이벤트는 다음에서 정의됩니다. BCOVIMAComponent.h파일.
  7. 샘플 앱에는 광고가 재생되는 동안 플레이어 컨트롤을 숨기고 광고가 끝나면 표시하는 기능이 포함되어 있습니다.

프리휠

Freewheel AdManager SDK는 CocoaPods를 사용하여 설치되지 않습니다. 대신 프로젝트의 애플리케이션 대상에 수동으로 추가해야 합니다.

FreeWheel 광고를 사용하려면 다음 단계를 따르십시오.

  1. FreeWheel AdMananger SDK를 프로젝트에 수동으로 추가합니다. 자세한 단계는 기본 프리휠 플레이어의읽어보기파일.
  2. 찾아 검토기본 FreeWheel 플레이어샘플 앱.
  3. 열기뷰컨트롤러파일.

  4. 귀하의 가치로 프로젝트를 사용자 정의하십시오. Video Cloud 라이브러리에서 콘텐츠를 검색하려면 다음 값이 필요합니다.

    • 재생 API 정책 키
    • 계정 ID
    • 동영상 ID
  5. FreeWheel AdManager를 만듭니다. 이것은 모든 광고 컨텍스트를 담당합니다.
  6. 광고 컨텍스트 정책을 사용하여 세션 공급자와 재생 컨트롤러를 만듭니다.
  7. 고유한 FreeWheel 구성 값으로 프로젝트를 사용자 정의하십시오. 인용하다 FreeWheel 문서또는 FreeWheel 계정 관리자에게 문의하십시오.
  8. 광고 보기, 광고 컨트롤 및 컴패니언 슬롯을 구성합니다.
