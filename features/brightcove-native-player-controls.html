소개

Brightcove Web Player 미학을 iOS 및 Android 플랫폼 표준과 결합한 Brightcove Native Player Controls는 즉시 사용할 수 있는 견고한 기능 세트 위에 세련된 모양을 제공합니다.

Brightcove 기본 플레이어 컨트롤의 기능:

  • 동적 UI
  • 커스터마이징
  • 접근성

동적 UI

Brightcove 네이티브 플레이어 컨트롤은 상황에 관계없이 콘텐츠가 아름다운 사용자 인터페이스에 표시되도록 적응력이 뛰어납니다.

적응형 화면 크기

UI 요소가 명확하게 보고 사용할 수 있을 만큼 충분히 큰지 확인하는 것은 모든 모바일 애플리케이션에서 중요합니다. Brightcove 기본 플레이어 컨트롤에는 사용자 인터페이스에 사용할 수 있는 공간을 결정하는 논리가 포함되어 있습니다. 너비가 특정 임계값 아래로 떨어지면 컨트롤이 자동으로 두 줄 구성을 사용하도록 전환됩니다.

기계적 인조 인간

Android용 네이티브 SDK는 장치 화면 크기를 확인하여 플레이어 컨트롤 레이아웃을 설정합니다.

  • 너비가480dp이상인 경우 한 줄 레이아웃이 사용됩니다.
  • 너비가 다음보다 작을 때480dp , 두 줄의 컴팩트 레이아웃이 사용됩니다.

밀도 독립 픽셀(dp)에 대해 자세히 알아보려면 Android의다양한 픽셀 밀도 지원문서.

iOS

iOS용 네이티브 SDK는 장치 화면 크기를 확인하여 플레이어 컨트롤 레이아웃을 설정합니다.

  • 너비가450px이상인 경우 한 줄 레이아웃이 사용됩니다.
  • 너비가 다음보다 작을 때450px , 두 줄의 컴팩트 레이아웃이 사용됩니다.

재생 모드

플레이어에는 수명 주기 동안 각각 특정 사용자 인터페이스 레이아웃이 필요한 다양한 유형의 콘텐츠가 포함될 수 있습니다. Brightcove 기본 플레이어 컨트롤에는 콘텐츠 유형이 VoD, Live 또는 Live DVR인지 자동으로 결정하고 그에 따라 사용자 인터페이스를 조정하는 로직이 포함되어 있으며, 콘텐츠 유형이 변경되는 재생 목록 전체에서도 이에 따라 사용자 인터페이스를 조정합니다!

주문형 비디오(VoD)

대부분의 재생 경험을 고려하면 VoD 인터페이스는 모든 플레이어 UI 디자인의 루트입니다.

아래 이미지를 스크롤하여 Brightcove 플레이어 컨트롤을 확인하십시오. 뷰어의 너비가 작을 때 컨트롤이 두 줄의 컴팩트 레이아웃으로 전환됩니다.

 

Brightcove Player Controls의 주목할만한 기능은 다음과 같습니다.

  • 재생/일시정지 버튼
    • 콘텐츠 재생 및 일시 중지
  • 뒤로 이동 버튼
    • 누르면 콘텐츠가 10초 동안 뒤로 검색됩니다.
  • 향상된 탐색 막대
    • 재생된 콘텐츠를 분홍색으로 표시
    • 밝은 회색으로 버퍼링된 콘텐츠를 표시합니다.
    • 광고 또는 기타 목적을 위한 마커 사용 지원(Android)
  • 자막 버튼
    • 캡션이 감지된 경우에만 표시됩니다. 이 버튼을 선택하면 자막 설정 메뉴가 나타납니다.
  • 전체 화면 버튼
    • 뷰어가 전체 화면으로 이동하거나 인라인 재생으로 돌아갈 수 있습니다.
  • 에어플레이 버튼(iOS만 해당)
    • Airplay 장치가 감지된 경우에만 표시됩니다. 이 기능을 통해 시청자는 Airplay 스트리밍을 설정할 수 있습니다.

살다

라이브 UI는 사용자가 참여할 수 없는 특정 요소를 숨깁니다. 제거된 요소에는 뒤로 이동 버튼과 탐색 막대가 포함됩니다. 그러나 녹색일 때 시청 중인 콘텐츠가 라이브임을 나타내는 새로운 "LIVE" 레이블이 추가됩니다.

라이브 DVR

라이브 DVR 형식은 VoD 레이아웃과 동일한 기능을 제공하며, 콘텐츠가 라이브일 때 녹색으로 바뀌고 시청자가 오래된 콘텐츠를 시청할 때 회색으로 바뀌는 추가 라이브 레이블을 제공합니다. 또한 라이브 레이블은 가장 최근 라이브 순간으로 돌아가기 위해 언제든지 누를 수 있는 버튼 역할을 합니다.

커스터마이징

앱의 색상 테마와 일치하도록 색상이나 스킨을 변경하고 싶으십니까? 문제가 아니다! Brightcove 기본 플레이어 컨트롤은 쉽게 사용자 지정할 수 있도록 설계되었습니다. 다음은 컨트롤을 사용자화할 수 있는 몇 가지 방법입니다.

  • 색상을 바꾸고 나만의 맞춤 버튼을 사용하여 나만의 테마를 만드세요. Android용 기본 SDK를 사용하면 다양한 공통 버튼을 쉽게 포함/제외할 수 있습니다. 여기에서 제어 버튼은 주황색으로 변경되었으며 탐색 막대에는 사용자 정의 이미지가 있습니다.
  • iOS용 기본 SDK를 사용하면 공통 버튼 및 레이아웃 집합에서 선택할 수도 있지만 사용자 지정 레이아웃을 만들 수도 있습니다. 다음은 사용자 정의 레이아웃의 예입니다.
  • 또는 버튼을 제거하여 간단하고 깔끔한 레이아웃을 만들 수 있습니다.

접근성

접근성 기능은 추가 설정이 필요 없이 기본적으로 UI의 일부로 제공됩니다.
